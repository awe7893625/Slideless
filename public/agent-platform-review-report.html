03 / 追加修復

Sprint 9 待辦 → 提前完成

4 個原本計劃留到 Sprint 9 的問題，本次一併修復。

#問題修復方式影響範圍
A Admin stats 無 RBAC routers/admin.py 加入 role=admin 檢查，非 admin → 403 安全性
B 密碼複雜度只檢查長度 schemas/auth.py 新增大寫 + 小寫 + 數字規則 帳號安全
C Webhook SSRF 用阻塞 DNS webhook_service.py 改 asyncio.getaddrinfo() async，不阻塞 event loop 效能 + 安全
D 部署說明複雜難用 README 重寫：Docker 一行部署 + Railway/Render/VPS 完整指南 開發者體驗

🔐 Admin RBAC

/api/platform/stats 現在要求 role=admin，一般用戶收到 403 Forbidden。已新增測試 test_admin_stats_non_admin_forbidden。

🔑 密碼複雜度

密碼規則升級：長度 ≥ 8 + 至少一個大寫 + 一個小寫 + 一個數字。測試密碼全面更新（TestPass1 等）。

🌐 Webhook 非阻塞 DNS

將 socket.gethostbyname()（阻塞）改為 asyncio.get_event_loop().getaddrinfo()，SSRF 防護不再影響 event loop 效能。

📦 一鍵部署

Docker Compose 一行指令搞定所有依賴（PG + Redis + API）。唯一需求：Docker Desktop。

04 / 部署簡化

部署方式：Before vs After

原本需要手動安裝 PostgreSQL + Redis + Python，現在一個 Docker 指令搞定。

❌ Before（複雜）

# 需要先裝 PostgreSQL 16 + Redis + Python
brew install postgresql@16 redis
createdb agentplatform
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app ...

✅ After（一鍵）

git clone <repo> ai-agent-platform
cd ai-agent-platform/infra
echo "SECRET_KEY=$(openssl rand -hex 32)" > .env
echo "POSTGRES_PASSWORD=agentpass" >> .env
docker compose up -d

完成後：後端啟動在 http://localhost:8000，API 文件在 http://localhost:8000/docs

05 / 雲端部署

雲端部署 選項比較

依據需求選擇最適合的部署方案。個人/小團隊推薦 Railway。

方案難度費用建議對象
Railway ⭐ 最簡單 免費起 / $5+ 個人 / 小團隊首選，自動提供 PG + Redis
Render ⭐⭐ 免費起 / $7+ 次選，免費方案有睡眠問題
VPS (Ubuntu) ⭐⭐⭐ $5–10/月 需要完整控制，懂 Linux
本機 Docker ⭐ 最簡單 免費 開發 / 測試環境
