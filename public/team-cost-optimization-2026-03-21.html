02 · Bug 修復

兩個系統性 Bug 已修復

這兩個 Bug 都會導致任務板爆炸式增長，已全部清除
BUG-01 · evaluation_feedback 暴風

評分觸發任務無上限

每個評分低於 4.0 的任務都會自動建立 Kai 任務，且沒有速率限制，導致任務板在短時間內產生大量改善任務，擠佔真實工作。

Before

評分 <4.0 → 自動建 Kai 任務，無上限、無過濾。improvement/feedback 來源的任務也會再觸發，形成遞迴。

→
After

加入 max 3 件/小時速率限制。跳過 source=improvement 或 source=feedback 的任務，斷開遞迴鏈。

BUG-02 · 遞迴任務鏈（3–4 層）

子任務不斷繁殖

dedup_window 設為 15 秒，比 retry backoff（30/60/120 秒）還短。每次 retry 都通過去重檢查，導致同一任務被建立 3–4 層子任務，任務板充斥殭屍任務。

Before

dedup_window = 15s，比 retry backoff 30s 短，每次 retry 都能通過去重。子任務可繼續生子任務，無深度限制。

→
After

dedup_window = 300s（5 分鐘），涵蓋完整 retry 週期。加入 chain depth guard：子任務不再允許建立下一層子任務。

03 · 成本優化

API 路由與模型重新配置

從 OpenRouter markup 走向零邊際成本訂閱制 + 直連 API
訂閱制成本模型說明

Claude Code Max 訂閱：claude-sonnet-4.6 無限制使用，透過 claw-claude-proxy（port 38780）讓 Kai backend 的所有 Claude call 免費。OpenRouter 有 markup，舊設定 FORCE_OPENROUTER=1 讓每個 Claude call 都有額外費用，移除後改走訂閱制，邊際成本歸零。

COST-01 · Claude 路由修正

移除 FORCE_OPENROUTER=1

Before

Kai backend 所有 Claude call 強制走 OpenRouter，每次都有額外 markup 費用。

→
After

改走 claw-claude-proxy（port 38780），使用 Claude Code Max 訂閱，零邊際成本。

COST-02 · Google API 直連

優先直連 Google API（新 Key 可用）

Before

llm_service.py 強制走 OpenRouter（因舊 Google key 被封），所有 Gemini call 都有 markup。

→
After

改為優先直連 Google API（新 key 可用），OpenRouter 降為 fallback。
