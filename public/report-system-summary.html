背景

任務起源

Rain 對報告系統的需求與 Klay 的工程化回應。

Rain 提供了一份 「AI 動態報告網站系統規範 v1.0」，描述了一套以 React + Framer Motion 為基礎的互動式報告系統構想。然而規範文件停留在設計層，缺乏可實際運行的工程實作。


Klay 接收任務後，將規範升級為 完整可執行的 Starter Kit，部署於 ~/Projects/ReportSystem/。目標是讓 Rain 未來製作任何報告頁面時，只需複製模板、替換資料，不再需要從頭建立動畫系統或圖表框架。


核心設計原則：Token 統一（動畫常數集中在 motion-tokens.ts，禁止 hardcode）、元件可復用（motion / chart / ui 三層獨立）、零配置可運行（npm install + npm run dev 即可預覽）。

使用指南

Rain 以後這樣用

六個步驟，從啟動到交付完整報告頁面。

1
啟動開發環境
進入專案目錄，安裝依賴並啟動 HMR 開發伺服器。
# 進入專案
cd ~/Projects/ReportSystem
# 安裝（首次）
npm install
# 啟動開發（localhost:5173）
npm run dev
2
複製報告模板
把 ReportPage.tsx 複製一份改名，作為新報告的起點。不要直接修改原模板。
cp src/pages/ReportPage.tsx src/pages/MyNewReport.tsx
3
替換資料
編輯 src/data/mock.ts，換上真實資料。格式已有 TypeScript 型別保護，填錯會有即時提示。
4
加動畫 — 用現成元件
所有動畫元件都在 src/components/motion/，直接包裹任何內容即可生效。
// 捲動進場
<FadeInSection direction="up" delay={0.2}>
  ...內容...
</FadeInSection>

// 子元素依序進場
<StaggerGroup>
  <StaggerItem>卡片一</StaggerItem>
  <StaggerItem>卡片二</StaggerItem>
</StaggerGroup>
5
加圖表 — 一律包在 ChartShell 內
ChartShell 提供統一的 title / loading / empty 三種狀態。圖表永遠透過 ChartShell 呈現。
<ChartShell title="月收入趨勢" source="Kai 中控台">
  <AnimatedLineChart data={revenueData} />
</ChartShell>
6
動畫數值從 Token 取，禁止 hardcode
所有 duration、easing 值從 motion-tokens.ts 匯入。這樣修改全站動畫節奏只需改一個地方。
import { DURATION, EASING } from '@/lib/motion-tokens'

// ✅ 正確
transition={{ duration: DURATION.base, ease: EASING.easeOut }}

// ❌ 禁止 hardcode
transition={{ duration: 0.4, ease: [0.25, 0.1, 0.25, 1] }}
