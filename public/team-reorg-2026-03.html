02 / 申請指南
4 個新整合 — 怎麼拿到 API Key
💬 Line Notify Token 免費
申請步驟
  1. 前往 notify-bot.line.me/my 並登入 Line 帳號
  2. 點選「Generate token」
  3. 命名 token（建議：Kai 通知）→ 選擇接收頻道（1 對 1 或群組）
  4. 複製生成的 token（只顯示一次，請立即存好）
# 加到 backend/.env
LINE_NOTIFY_TOKEN=your_token_here
費用：完全免費，無流量限制
📱 Twilio SMS 試用 $15 免費
申請步驟
  1. 前往 twilio.com/try-twilio 註冊免費試用帳號
  2. 進入 Console → 取得 Account SID + Auth Token
  3. 取得測試發送號碼（免費帳號提供 1 組）
  4. 驗證要發送的目標號碼（試用帳號限制，需先驗證接收方）
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your_auth_token
TWILIO_FROM=+1XXXXXXXXXX
費用：試用帳號含 $15 美元額度，之後 SMS 約 $0.0079 / 則（台灣國際簡訊略高）
🐵 Mailchimp 500 聯絡人免費
申請步驟
  1. 前往 mailchimp.com 選擇免費方案（500 聯絡人 / 1000 封/月）
  2. 完成帳號設定後，進入 Account → Extras → API Keys
  3. 點選「Create A Key」→ 複製 API Key
  4. 注意 Key 尾端的 -us1（server prefix），不同帳號可能是 us2、us6 等
MAILCHIMP_API_KEY=xxxxxxxxxxxxxxxx-us1
# list_id 在 Audience → Settings → Audience name and defaults 找到
費用：免費方案（500 聯絡人 / 月 1000 封）對中小型業務完全夠用
💳 Stripe 按交易收費
申請步驟
  1. 前往 stripe.com（台灣可申請，需填公司或個人資料）
  2. 完成身份驗證後，進入 Dashboard → Developers → API Keys
  3. 取得 Secret key（sk_live_xxx 正式環境 / sk_test_xxx 測試）
  4. Publishable key（pk_live_xxx）供前端使用
STRIPE_SECRET_KEY=sk_live_xxxxxxxxxxxxx
STRIPE_PUBLISHABLE_KEY=pk_live_xxxxxxxxxxxxx
費用：每筆 2.9% + $0.30 USD（無月費，無固定成本）
03 / 串聯邏輯
Workflow 的思考方式
核心概念
Kai Flows 的每個 step 都可以把輸出存到變數，下一個 step 用 {{變數名}} 引用。這樣就能把資料從一個 action 傳到下一個，形成完整的自動化管道。每個 step 的 id 就是變數名稱，用 {{step_id.data}} 或 {{step_id.output}} 取得結果。
三段式管道架構
觸發 / 資料來源
RSS
Stripe 訂單
表單資料
→
處理 / 判斷
condition
set_var
agent（AI）
→
通知 / 記錄
Discord + Line
Mailchimp + Email
Notion + SMS
變數傳遞範例
# Step 1 的 id 是 "fetch_data"，結果存在 {{fetch_data.data}}
steps:
  - id: fetch_data
    action: http_request
    url: https://api.example.com/data

# Step 2 用 {{fetch_data.data}} 取得上一步的結果
  - id: process
    action: agent
    prompt: "分析這份資料：{{fetch_data.data}}"

# Step 3 用 {{process.output}} 取得 AI 分析結果
  - id: notify
    action: line
    message: "分析結果：{{process.output}}"
04 / 電商應用
TheOrder — 訂單自動化
新訂單 → Line + SMS 雙通知
Order Notify Workflow
name: order_notify
steps:
  - id: fetch_order
    action: http_request
    method: GET
    url: https://theorder.tw/api/store/orders

  - id: notify_line
    action: line
    token: "{{env.LINE_NOTIFY_TOKEN}}"
    message: "🛍 新訂單！{{fetch_order.data.0.id}}"

  - id: notify_sms
    action: twilio
    to: "+886912345678"
    body: "序位電商：收到新訂單，請確認出貨"
顧客下單 → 自動加入 Mailchimp 行銷名單
Customer Subscribe Workflow
name: customer_subscribe
steps:
  - id: add_to_list
    action: mailchimp
    op: add_member
    list_id: "{{env.MC_LIST_ID}}"
    email: "{{trigger.customer_email}}"
    first_name: "{{trigger.customer_name}}"
    tags: ["theorder", "新顧客"]

  - id: confirm_discord
    action: discord
    message: "✅ 顧客 {{trigger.customer_email}} 已加入行銷名單"
月底 Stripe 收款 → AI 對帳報告
Monthly Revenue Workflow
name: monthly_revenue
steps:
  - id: fetch_payments
    action: stripe
    op: list_payments
    limit: 100

  - id: report
    action: agent
    prompt: "分析這些 Stripe 付款記錄，摘要本月收入：{{fetch_payments.data}}"

  - id: send_report
    action: email
    to: "asdapper8866@gmail.com"
    subject: "月收入報告"
    body: "{{report.output}}"

06 / 團隊應用
Kai 內部 — AI 團隊自動化
每日 AI 新聞 → 自動摘要 → Line 推播
AI News Brief — cron: 0 8 * * *（每天 08:00）
name: ai_news_brief
# cron: 0 8 * * * (每天早上 8 點)
steps:
  - id: fetch_news
    action: rss
    url: https://feeds.feedburner.com/techcrunch
    limit: 5

  - id: summarize
    action: agent
    prompt: "用繁體中文摘要這 5 則 AI 新聞，每則 2 句：{{fetch_news.items}}"

  - id: push_line
    action: line
    message: "📰 今日 AI 快訊\n{{summarize.output}}"
每日任務完成摘要 → Notion 日誌
Daily Task Log — cron: 0 23 * * *（每天 23:00）
name: daily_task_log
# cron: 0 23 * * * (每天 23:00)
steps:
  - id: fetch_tasks
    action: http_request
    method: GET
    url: http://127.0.0.1:5002/api/tasks?status=done

  - id: summarize
    action: agent
    prompt: "摘要今日完成的任務：{{fetch_tasks.data}}"

  - id: save_notion
    action: notion
    op: create_page
    database_id: "{{env.NOTION_LOG_DB}}"
    properties:
      日期: "today"
      摘要: "{{summarize.output}}"
