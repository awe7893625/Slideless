01 · 任務起點

問題：AI Agent 跑了什麼？沒人知道

Rain AI Team 每天執行數十個 Kai 任務，但完全缺乏可觀測性 —— 不知道每個任務花了多少錢、哪個環節最慢、哪個 Model 最貴、哪個 Agent 最耗資源。
問題

零 Observability

任務完成後沒有任何 trace 記錄，無法知道 LLM 呼叫了幾次、用了哪個 model、耗費多少 token

問題

成本盲區

Claude Opus vs Haiku 成本差 50x，但沒有任何計費追蹤，無法優化 model 選擇，月費失控

目標

自動追蹤每個任務

每個 Kai 任務完成後自動記錄 trace：成本、延遲、每個 subagent 的個別表現

Ray 企劃書 — Route B（Observability）

Tier 1 升級三條路中，ROI 最高的首選

Route A: Graph Engine（高成本，3-4 週）｜Route B: LangFuse Observability（低成本，1-2 天）｜Route C: Evaluation Pipeline（需先有 B）

Ray 判斷：Route B 是先決條件 —— 沒有 observability 數據，就無法做 Route C 的 Evaluation；也無法量化 Route A 的效能改善。先建數據基礎，後做一切優化。

02 · 技術障礙與解法

四道連環障礙，逐一攻破

從環境到 SDK 一路踩坑，每個障礙都有明確根因與解法。
障礙 1 — 環境
Docker Desktop 損壞：no space left + 執行檔遺失
Docker daemon 無法啟動，disk 使用率過高且執行檔損壞。解法：移除 Docker Desktop，改用 Colima（--cpu 2 --memory 4 --disk 20），完全相容 Docker CLI，開機自動啟動
障礙 2 — 版本
LangFuse v3 需要 ClickHouse 額外依賴
v3 引入 ClickHouse 作為 OLAP 儲存，自架成本大增。解法：降版至 v2（tag: langfuse/langfuse:2），只需 PostgreSQL 16，docker-compose 30 行搞定
障礙 3 — 帳號
LangFuse v2 不支援 LANGFUSE_INIT_* 環境變數
v3 支援的初始化帳號環境變數，v2 完全沒有。解法：Playwright 自動化建立帳號，取得真實 API Key（已存入 .env，不公開）
障礙 4 — SDK
Python SDK v4 API 不相容：__init__() unexpected keyword
SDK v4 breaking change，所有初始化參數名稱改動。解法：捨棄 SDK 完全不用，改用 urllib.request 純 HTTP REST，永久免疫 SDK 版本漂移

04 · Trace 數據模型

每條 Trace 包含什麼數據

完整的三層數據模型：任務層級 trace → Generation spans（per subagent）→ Token usage observation
Layer 1 — Trace

任務層級

  • id 任務 UUID（與 Kai 一致）
  • name 任務標題
  • userId assigned agent
  • input 任務描述
  • output 結果摘要
  • tags [agent, task_type, source]
Layer 1 — Trace Metadata

成本與效能

  • cost_usd USD 成本（精確）
  • token_input / token_output
  • duration_seconds 執行時間
  • agent_model 使用的 model
  • latency_breakdown_ms 各 agent step 延遲
  • priority / severity_level / verified
Layer 2 — Generation Spans

每個 agent_step

  • span_type llm / tool / retrieval / agent
  • model 該 step 的 model
  • input_summary / output_summary
  • latency_ms 個別延遲
  • cost_usd 個別成本
  • tool_name（tool span）/ retrieval_source（retrieval span）
span_type 分類系統（新增）

四種 span 類型，對應不同 AI 操作

span_type觸發場景關鍵 metadata 欄位LangFuse 視覺化
llmClaude/GPT/Gemini 直接呼叫model, latency_ms, cost_usdGeneration — 顯示 token usage
toolBash / Write / Grep / WebFetch 等工具執行tool_name, latency_msSpan — 工具名稱標籤
retrievalMemory / RAG / Vector DB 查詢retrieval_source, latency_msSpan — 查詢來源標籤
agent子 Agent 呼叫（如 Klay 呼叫 John）model, cost_usd, latency_msGeneration — 嵌套 trace
# agent_steps 範例（force_complete 時傳入） agent_steps = [ { "agent_name": "max", "span_type": "llm", # llm | tool | retrieval | agent "model": "claude-opus-4-6", "token_input": 2000, "token_output": 800, "latency_ms": 3500, "input_summary": "設計 DB schema", "output_summary": "完成 ERD 設計" }, { "agent_name": "john", "span_type": "tool", # Bash / Write / Grep 工具執行 "tool_name": "Write", # tool span 專用欄位 "latency_ms": 450 } ]
