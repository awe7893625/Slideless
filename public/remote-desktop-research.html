Executive Decision

結論：RustDesk 自架 + AI Agent 整合

主系統：RustDesk Server（自架） — 完全控制、可整合 AI、免授權費、可商業化。

輔助系統：TeamViewer（過渡期人工接管） — 在自建系統成熟前，人工支援場景使用。

AI 控制核心：Browser Agent（Playwright）+ System-level（AppleScript/PowerShell）

為什麼選 RustDesk？ 唯一同時滿足：自架 Server + 開源可改 + P2P 穿透 + 跨平台 + AI 可整合。 MeshCentral 也可自架但缺少 P2P（依賴 relay）；TeamViewer/AnyDesk 無法 AI 控制且會封商用。

架構設計原則

  • 1. AI-First — 所有操作都可由 AI Agent 自動執行，人工介入是備用
  • 2. Self-Hosted — RustDesk Server 自架，不依賴第三方 SaaS，不會被封鎖
  • 3. 三層控制 — Browser Agent（最穩）> System Agent（最強）> Vision Agent（最通用）
  • 4. 可商業化 — AGPL-3.0 允許 SaaS 模式（不分發 binary），或購買 RustDesk Pro
Section 02

AI 控制方式（核心）

這才是整個系統的關鍵：AI 怎麼操作遠端電腦。三種方式，依場景選用。

推薦 | 最穩定 方式一：Browser Agent（Playwright）

透過遠端瀏覽器操作網頁應用。最適合商業化 — 90% 的客戶需求都在瀏覽器裡完成。

技術Playwright MCP / Chrome DevTools Protocol（CDP）
能力點擊、輸入、截圖、表單填寫、檔案上傳、多分頁管理
適用SaaS 操作、網站部署、CMS 設定、後台管理、線上服務申請
優點穩定可靠、有結構化 DOM、可精確定位元素、失敗可重試
限制只能操作瀏覽器內的東西，無法控制桌面應用
現有整合Kai 已有 Playwright MCP + Chrome MCP，可直接複用
AI Agent → RustDesk 連線 → 遠端 Chrome → Playwright CDP → 操作網頁
                                        ↑
                              Client PC 上的 Chrome
                              開啟 --remote-debugging-port

最強大 方式二：System-Level Agent（OS 原生指令）

直接透過 OS 指令操作桌面應用。能做到瀏覽器做不到的事：安裝軟體、設定系統、管理檔案。

macOSAppleScript + osascript — 操作 Finder、System Preferences、任何 app
WindowsPowerShell + COM 自動化 — 操作任何 Windows 應用、Registry、服務
Linuxxdotool + xclip + CLI — 模擬滑鼠鍵盤、管理套件
適用安裝軟體、系統設定、檔案管理、CLI 操作、桌面 App 控制
優點控制力最強，能做任何人能做的事
限制跨平台需分別維護腳本，錯誤處理較複雜
AI Agent → RustDesk SSH/指令通道 → Client OS
                                              ├── macOS: osascript "tell app Finder to ..."
                                              ├── Windows: PowerShell Install-Module ...
                                              └── Linux: bash apt install ...

最通用 方式三：Screenshot + Vision AI（視覺操控）

截圖 → GPT-4V/Claude Vision 識別畫面 → 模擬滑鼠點擊。萬能備用，什麼都能操作。

技術Screenshot API + Vision LLM（GPT-4V / Claude Vision）+ 滑鼠模擬
流程截圖 → AI 看圖理解 → 決定點哪裡 → 發送座標 → 驗證結果
適用無法用前兩種方式的場景：遊戲、特殊桌面 App、驗證碼
優點最通用，任何 UI 都能操作，不需要 API 或結構化資訊
限制速度慢（每步 2-5 秒）、Vision API 成本高、準確度較低
類似產品Anthropic Computer Use、OpenAI Operator、OpenClaw
AI Agent → RustDesk 截圖 → Vision AI 分析 → 座標計算 → 模擬點擊
     ↑                                                                    │
     └────────────── 驗證截圖（確認操作成功） ◄──────────────────────────┘

AGPL-3.0 授權注意事項

RustDesk 是 AGPL-3.0。關鍵規則：如果你修改了 RustDesk 原始碼並以 SaaS 方式提供服務， 修改的部分需要開源。兩個解法：

  • 方案 A（推薦）：不改 RustDesk 核心，只在外層包裝 AI Agent 層。 RustDesk 當純粹的遠端通道，AI 邏輯獨立 — 不觸發 AGPL
  • 方案 B：購買 RustDesk Pro 授權（$19.99/月起），獲得商業許可，可閉源修改
  • 方案 C（長期）：參考 RustDesk 架構自建（不 fork），用 Apache 2.0 自有協議
Section 04

商業風險與對策

遠端控制 = 高風險領域，必須提前處理。

風險 1：TeamViewer 封鎖商用帳號

免費版偵測到頻繁商業使用會鎖帳號，斷線後無法連回。已有大量用戶遇到。

對策：TeamViewer 僅作過渡期備用，主力用自架 RustDesk，完全不受封鎖影響。

風險 2：AGPL 授權傳染

修改 RustDesk 核心程式碼，衍生作品需開源。若不小心觸發，自有商業機密可能外洩。

對策：方案 A — AI 層與 RustDesk 隔離（不改核心）。或方案 B — 購買 Pro 授權。

風險 3：客戶資安疑慮

「讓 AI 控制我的電腦」心理障礙高。資料外洩、誤操作都是客戶擔心的。

對策：全程錄影 + 操作日誌 + 客戶即時觀看畫面 + 一鍵中斷按鈕 + 書面授權同意書。

風險 4：法規合規（個資）

操作客戶電腦可能接觸個資，台灣個資法 + GDPR（如有歐洲客戶）需遵守。

對策：服務條款 + 資料處理協議（DPA）+ 操作範圍明確限定 + 操作結束清除快取。

風險 5：AI 誤操作

AI 判斷錯誤執行了不該做的操作（刪檔、改設定、送出表單）。

對策：高風險操作需人工確認（Exclusion Rules）+ 操作前快照 + 可回滾機制。

風險 6：NAT 穿透失敗率

對稱型 NAT 環境下 P2P 穿透失敗，需 Relay 中繼增加延遲和流量成本。

對策：自架 coturn TURN Server 作為 Relay fallback，確保 100% 連線率。

開發路線圖

三軌並行：TeamViewer 立即可用 → RustDesk 自架 → AI 整合

Phase 0：立即可用（現在）

Week 0 — 已完成

TeamViewer 人工控制已就緒。CLI 工具、設備註冊、客戶安裝引導頁、Doctor 診斷全部到位。
產出：teamviewer-team CLI、remote-setup.html

Phase 1：RustDesk Server 自架

Week 1-2

在 Rain 的 Mac 或 VPS 上部署 RustDesk hbbs（信令）+ hbbr（中繼）Server。 設定自訂 ID Server。建立 Client 安裝腳本（一鍵安裝 + 指向自架 Server）。
產出：RustDesk Server 運行、Client 安裝包、連線測試通過

Phase 2：AI + Browser Agent 整合

Week 3-5

透過 RustDesk 連線後，在遠端啟動 Chrome（--remote-debugging-port）， Playwright CDP 連入操作。Kai 中控台新增「遠端作業」面板。
產出：AI 可自動操作遠端瀏覽器、操作日誌記錄

Phase 3：System Agent + Vision Agent

Week 6-9

加入 AppleScript/PowerShell 系統級控制。 加入 Screenshot + Vision AI 作為萬能 fallback。 三種控制方式的自動切換邏輯。
產出：完整三層控制、智慧切換

Phase 4：客戶端產品化

Week 10-14

Client 安裝程式品牌化（自有 Logo + 名稱）。 客戶管理面板（設備清單、連線狀態、操作紀錄）。 服務條款 + 授權同意流程。 定價模型設計。
產出：可對外銷售的 v1.0 產品

Phase 5：商業化 + 規模化

Week 15+

多租戶支援（不同客戶隔離）。 API 對外開放（讓其他 AI 團隊串接）。 考慮從 AGPL RustDesk 遷移至自建 Rust P2P Client。
產出：SaaS 平台、API 文件、商業授權
