  • 「如果不裝這些 skills，Rain AI Team 會有什麼損失？」 — 損失有限。真正填補空缺的只有 3 個：api-test-suite-builder（無系統化 API 測試）、performance-profiler（無正式效能剖析流程）、mcp-server-builder（未來 MCP 擴展需求）。

05 — 優先使用

Ray 推薦的 Top 3 立即啟動

這 3 個填補了 Rain AI Team 目前最明顯的空缺，且技術棧完全符合，立即可用。

1
api-test-suite-builder → Max / Rex
Kai API（FastAPI :5002）和 TheOrder API（Next.js App Router）都缺乏系統化的 auth/input validation 測試。注意：需分離 /api/admin/ 和 public routes 的測試環境（admin JWT 問題）。
2
performance-profiler → Max
TheOrder product page 有 SSR，市場晨報腳本跑多個 HTTP fetch，都是候選目標。注意：py-spy 在 macOS ARM 需 sudo，改用 --nonblocking 模式可避免。
3
monorepo-navigator → Forge / John
TheOrder 是 pnpm + Turborepo monorepo，CI 改用 --filter=...[HEAD~1] 取代全量 rebuild，立即改善 Forge CI 效率。
