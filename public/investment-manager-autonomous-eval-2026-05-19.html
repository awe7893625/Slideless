422 觸發情境
三種拒絕場景
1. 無 decision_log 欄位
2. decision_log 非合法 JSON
3. reasoning 欄位為空字串

換交易動作（buy→exit）必須附新理由，不接受沿用舊 reasoning（W6）
場景 請求內容 回應 測試案例
buy 無 decision_log POST /ledger action=buy HTTP 422 test_buy_no_reasoning
buy 有合法 reasoning POST /ledger action=buy + log HTTP 201 test_buy_with_reasoning
PATCH 關倉無理由 PATCH /ledger/{id} status=closed HTTP 422 test_patch_close_no_reasoning
buy→exit 沿用舊 log PATCH action=exit + 原 reasoning HTTP 422 test_exit_reuse_old_reasoning
DELETE hard DELETE /ledger/{id} 200（繞過） admin 逃生口 by design

reasoning非空字串（強制）
