一、Wilson Huang 核心觀點提煉Rain AI Team Optimization Report　2026-03-12
內容分析

他在關注什麼：三組主題

主題 A — Nat Eliason 的 Felix 模型（月賺 $80K，成本 $1,500）
核心模型：Nat Eliason 用 Claude 建了一個叫 Felix 的 AI Agent，底下有兩個 sub-agent（客服 + 銷售），Felix 每晚自動 review 當天所有對話，找一個可以改進的地方，然後自己修改自己的 system prompt。60 天的 1% 複利改善，人類只負責透過 Discord 溝通，不寫一行 code。

架構關鍵點

  • Sub-agent 分工：客服 Agent + 銷售 Agent，各有明確角色
  • Nightly cron job：每晚自動 review + 自我優化
  • 1% 複利：60 天 × 1% = 79% 改善（e^0.6 ≈ 1.82x）
  • Discord 介面：人類與 Agent 的唯一接觸點

商業數字

  • 月營收：$80,000+
  • 月成本：$1,500（API + infra）
  • 毛利率：~98%
  • 人力投入：每天零碎時間，有全職工作
  • Crypto 角色：Agent 間支付、micropayment、身份驗證
主題 B — Heartbeat Protocol（取代 Agile 站會）
Tremaine Grant 提出：Agent 時代兩週一個 sprint 太慢。改用每小時一次 telemetry check——自動報告 Agent 的執行狀態，取代人工站會。同時給每個 Agent 明確的角色定位 + 一個「北極星目標」，再加一個專門唱反調的 Skeptic Agent，整體輸出品質大幅提升。

人工站會（15-30 分鐘）自動摘要推送 Discord

批次回顧即時異常告警
主題 C — a16z OODA Loop 媒體策略
OODA Loop（觀察→定向→決策→行動）：速度本身就是優勢。「有趣比無害重要」，企業品牌時代結束，人格化品牌才是通行證。創辦人親自出來講話，比任何公關稿都有效。短推文 = 口語文化；長 podcast = 書寫文化，平台特性決定內容策略。

Wilson 的底層思維模型

他在押注的方向

  • AI Agent 經濟：人類 → 指揮者，AI → 執行者
  • Crypto 是 Agent 間支付的基礎層
  • 自我優化 Agent 比固定規則 Agent 更有複利效應
  • 速度 > 完美（OODA Loop 勝在反應速度）

對我們的啟示

  • 我們的 Agent 有沒有自我優化機制？
  • 我們的 Kai 監控是否達到每小時心跳？
  • 我們有沒有Skeptic Agent角色？
  • Rain 本人的人格化品牌在哪？

三、五大優化方向Rain AI Team Optimization Report　2026-03-12

五大優化方向（依優先級排序）

P0 優化 1：實作 Nightly Self-Optimization Loop

「Felix 每晚透過 cron job 回顧當天所有對話，找出一個可以改進的地方，然後自己修改自己的系統」— Wilson Huang

現況痛點

  • lessons.md 靠 Klay 手動更新，常常忘記
  • 相同的錯誤一再發生（signal 沒有 feedback loop）
  • Session 結束後洞察消失在 context 裡

改進方案

  • 每個 session 結束 → hook 自動 append 錯誤 + 修正到 lessons.md
  • 每週一 Klay 讀 lessons.md → 更新 CLAUDE.md + 各 Agent SKILL.md
  • 負責人：Kai（監控）+ Klay（週回顧）

P0 優化 2：Heartbeat Protocol — 每小時 Agent 狀態推送

現況痛點

  • 不知道 background task 是否還在跑
  • Agent 卡住或失敗，Rain 要主動問才知道
  • 目前只有 08:00/20:00 市場晨報，無 Agent 心跳

改進方案

  • Kai 每小時 → 掃描 active tasks → 推送 Discord #agent-heartbeat
  • 格式：哪些在跑、哪些卡住、哪些完成（3 秒能讀完）
  • 卡住超過 30 分鐘 → 自動 ping Rain + 嘗試 restart
  • 負責人：Kai（新增 heartbeat API）

P1 優化 3：制度化 Skeptic Review Pass

「給每個 Agent 明確的角色定位和一個北極星目標，再加上一個專門唱反調的 Skeptic Agent，整體產出品質會大幅提升」

現況

  • Ray 有 veto 權，但定位是「設計審查」不是 Skeptic
  • Major 任務 Phase 3 沒有強制「最壞情況」分析
  • Vick 是備援角色，沒有常態觸發

改進方案

  • Ray Phase 3 新增必填欄位：「最壞情況 + 如何復原」
  • Major 任務 Scout 輸出後，強制跑一次 Vick 輕量 Skeptic review
  • Skeptic 問題模板：「如果這個假設是錯的？」「3 個月後最可能的失敗點？」
  • 負責人：更新 task-dispatch.md 加入 Skeptic 步驟

P1 優化 4：每個 Agent 設定北極星 KPI

現況

  • Agent 有角色定義，但沒有「這個 Agent 做好的衡量標準」
  • 很難判斷某個 Agent 是否值得繼續使用

改進方案（範例）

  • Quill：每週 PR review 率 ≥ 90%，退回率有效（≤ 30% 重做）
  • Forge：部署成功率 ≥ 99%，CI 失敗→修復 < 30 分鐘
  • Scout：架構規格書被實作端 採用率 ≥ 80%（不需大幅修改）
  • Kai：系統 uptime 報告準確率 ≥ 99.5%
  • 更新各 Agent SKILL.md 加一行北極星指標

P2 優化 5：Rain 人格化品牌語氣系統

現況

  • Buzz/EchoMarketer 產出的內容是「AI 感」，缺乏 Rain 個人語氣
  • a16z 的觀點：人格化品牌才是長期護城河
  • Wilson 本人就是品牌，不是公司名

改進方案

  • 建立 rain-voice.md：Rain 的寫作風格、慣用詞、觀點立場
  • EchoMarketer/Buzz 所有輸出必須過 rain-voice.md 校準
  • 每季更新一次 voice guide（根據 Rain 實際發過的文章）
  • 負責人：Noel（整理）+ EchoMarketer（執行）

核心結論

我們的優勢

  • Agent 廣度（29 個）遠超業界平均
  • 角色分工、審查機制、CI/CD 已成體系
  • 3-Phase Planning 已比 Felix 更系統化

我們的機會

  • 自動化程度不夠：仍有太多「人工觸發」
  • Agent 不會自己進步（無 self-optimization loop）
  • 監控粒度不夠細（heartbeat 缺失）
  • Rain 個人品牌聲音缺席於輸出內容
一句話總結：我們比 Nat Eliason 的 Felix 模型有更多 Agent、更好的分工，但 Felix 會自己進步而我們的 Agent 不會。下一階段的核心目標是讓 Rain 的 AI 團隊具備自我優化能力，而不只是執行能力。
