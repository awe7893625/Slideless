Domain Checklist

5 種任務類型的專屬審查清單 · Ray Phase 3 強制套用

根據任務的技術域（Website / Web App / Backend API / AI System / Crawler），套用對應的專屬審查 checklist。觸發條件：Ray Phase 3 審查（Major/Critical）。

Website

網站類型

  • SEO meta（title/desc/H1 唯一）
  • OG 標籤完整
  • Mobile RWD 驗收
  • GA4 追蹤安裝
  • JSON-LD Schema 正確
  • Core Web Vitals 基線
  • robots.txt / sitemap.xml
Web App

Web App 類型

  • Auth/Authz 邊界清楚
  • Rate limit 保護
  • CSRF 防護
  • Session 安全
  • Input validation 完整
  • Error message 不洩漏
  • 敏感操作二次確認
Backend API

後端 API 類型

  • API 文件（OpenAPI）
  • Health check endpoint
  • 結構化 Logging
  • Timeout/Retry 機制
  • Config separation
  • Error response 標準化
  • Pagination 支援
AI System

AI 系統類型

  • Prompt 版本化
  • Fallback model 設定
  • Token limit 保護
  • Cost monitoring
  • Tool failure recovery
  • Prompt injection 防護
  • 輸出格式驗證
Crawler

爬蟲類型

  • Rate control（請求間隔）
  • Retry + backoff 策略
  • Anti-bot 規避方案
  • 資料驗證（Schema）
  • Deduplication 機制
  • 失敗率告警閾值

Section 05 · 整合項 2（已完成）

Runtime Watch

Critical 任務部署後 1 小時確認機制 · Owner：Forge

新增到 Forge Evidence Gate（Critical 任務），確保部署後核心路徑正常運作。

確認項目

  • 核心路徑 HTTP 200（≥3 條端點）
  • 無 500 error log
  • Discord 發送 ✅ 完成確認

執行規則

  • 觸發：Critical 任務部署後
  • 窗口：部署後 1 小時內
  • Owner：Forge（明確，不可推諉）
  • 未確認 = 任務未結案
