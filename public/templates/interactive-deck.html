如何設定 API 刷新端點

在 JavaScript 的 REFRESH_CONFIG 物件修改：

  • 將 apiUrl 改為 Kai API 的實際端點（如 http://127.0.0.1:5002/api/tasks/stats）
  • 修改 mapData(data) 函數，將 API 回傳值對應到 metric 欄位
  • 如果沒有 API：設定 enabled: false，刷新按鈕會自動隱藏
05 · 元件說明
給 Nova 的使用說明

製作報告時，複製對應的 HTML 區塊套用即可。設計 token 已統一定義在 CSS :root。

圖表元件（Chart.js）

複製 .chart-card 區塊，修改 data-chart ID 和 canvas ID 保持一致。 在 JS 的 CHART_DATA 物件新增對應的 week/month/total 資料集。 Chart.js 4 從 CDN 載入，無需 npm。

Accordion 元件

複製 .accordion 區塊，修改 data-accordion 和對應 id 保持一致。預設收合，適合放原始 JSON、SQL 查詢結果、詳細 log。 不需要任何 JS 初始化，自動綁定事件。

Tab 切換元件

複製 .tabs 區塊，修改 data-tab 和對應 panel 的 id（格式：tab-{name}）。 支援任意數量的 Tab，不超過 6 個最佳。

刷新按鈕元件

修改 REFRESH_CONFIG.apiUrl 指向實際 Kai API 端點。 如無 API，設定 enabled: false 自動隱藏按鈕。 回傳資料格式對應 mapData() 函數自行調整。
