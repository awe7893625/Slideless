0 研究背景與摘要
本報告目標、研究範圍、核心問題

Rain 正在設計 AgentDesk 平台，讓不同電腦上的 AI Agent 團隊協作。核心問題是：如何實現 Agent-to-Agent 的任務分派、身份驗證、狀態同步？本報告研究了 Google A2A Protocol v1.0、AGNTCY（Cisco）、Clawdentity（IETF Draft）、 Hyperspace AGI、SPIFFE/SPIRE 等業界最新方案， 並針對 AgentDesk 私有部署場景提出具體架構建議。


研究結論明確：AgentDesk 不應採用純 P2P 架構。GossipSub/DHT 是公開去中心化網路的解法， 私有受信任環境應選擇 Hub-and-Spoke，以 Kai 作為中央 Hub，實現最簡單、最安全、 最可觀測的架構。安全層面採用 Ed25519 業界標準，MVP 用 JWT，Production 升級 mTLS。

1 核心結論
三個關鍵決策方向，直接影響 AgentDesk 設計
🚫
架構決策

不要用純 P2P

GossipSub / DHT 是公開去中心化網路設計的技術。AgentDesk 是私有受信任環境， 採用 Hub-and-Spoke（Kai 當 Hub）更簡單、安全、可觀測。 純 P2P 帶來不必要的複雜度，且缺乏集中控制與審計能力。

✅
協議選型

採用 Google A2A Protocol 格式

2025 年推出的業界標準（v1.0），Agent Card + Task lifecycle 設計成熟。 和現有 Kai task board 可直接對齊，避免重複造輪子。 跨框架互操作性強，未來可對接外部 Agent 生態系。

🔐
安全標準

Ed25519 是業界標準

IETF Clawdentity Draft 已確立 Ed25519 為 Agent 身份標準，Hyperspace 實作印證可行性。 MVP 用 JWT（實作快），Production 用 mTLS（Hub 作 CA，24h 憑證自動輪替）。

6 分階段實作建議
三個階段漸進演進，從 MVP 到生產級分散式架構
1

MVP — 立即可做

HTTP + JWT + PostgreSQL
  • Private Registry（PostgreSQL）
  • 欄位：agent_id, name, public_key, skills[], status, last_heartbeat
  • Agent Card JSON（A2A 規範格式）
  • Ed25519 keypair per-Agent
  • JWT 驗證（HS256，24h 過期）
  • Heartbeat 每 30s，60s 無心跳 → offline
  • Task 狀態機（submitted → working → completed/failed）
  • SSE 即時狀態推送
2

Production — 3~6 個月後

NATS JetStream + mTLS
  • NATS JetStream（subject-per-skill 路由）
  • mTLS（Hub 作為 CA）
  • Agent 憑證 24h 自動輪替
  • Circuit Breaker（失敗率 > 50% → 暫停路由）
  • Saga Pattern（長任務補償回滾）
  • 消息持久化（JetStream Stream）
  • Dead Letter Queue（失敗任務重試）
3

Scale — 長期目標

SPIFFE/SPIRE + Federation
  • SPIFFE/SPIRE（workload identity 自動輪替）
  • OPA（Open Policy Agent，細粒度授權）
  • Federation（多 Hub 跨組織協作）
  • Distributed Tracing（OpenTelemetry）
  • Rate Limiting per Agent per Skill
  • Multi-region 部署
7 關鍵資料模型
三張核心資料表的 PostgreSQL Schema 設計
agents — Agent Registry
CREATE TABLE agents (
  id           UUID        PRIMARY KEY
                            DEFAULT gen_random_uuid(),
  name         VARCHAR(64) UNIQUE NOT NULL,
  public_key   TEXT        NOT NULL,
  -- Ed25519 公鑰（Base64）
  skills       TEXT[]      NOT NULL DEFAULT '{}',
  machine_id   VARCHAR(64),
  -- 部署機器識別
  status       VARCHAR(16) NOT NULL DEFAULT 'offline',
  -- online / offline / busy
  last_heartbeat TIMESTAMPTZ,
  agent_card   JSONB,
  -- A2A Agent Card JSON
  created_at   TIMESTAMPTZ DEFAULT now()
);
tasks — Task Board
CREATE TABLE tasks (
  id           UUID        PRIMARY KEY
                            DEFAULT gen_random_uuid(),
  title        TEXT        NOT NULL,
  description  TEXT,
  status       VARCHAR(20) NOT NULL
                            DEFAULT 'submitted',
  -- submitted/working/input-required
  -- /completed/failed/canceled
  assigned_to  UUID REFERENCES agents(id),
  created_by   UUID REFERENCES agents(id),
  required_skill VARCHAR(64),
  priority     INT         DEFAULT 5,
  progress     INT         DEFAULT 0,
  result       JSONB,
  created_at   TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ
);
task_events — Event Log
CREATE TABLE task_events (
  id           UUID        PRIMARY KEY
                            DEFAULT gen_random_uuid(),
  task_id      UUID        NOT NULL
                 REFERENCES tasks(id)
                 ON DELETE CASCADE,
  agent_id     UUID        REFERENCES agents(id),
  event_type   VARCHAR(32) NOT NULL,
  -- status_change / progress / message
  -- / artifact / error
  payload      JSONB,
  created_at   TIMESTAMPTZ DEFAULT now()
);

-- Index for SSE stream queries
CREATE INDEX idx_task_events_task
  ON task_events(task_id, created_at DESC);
8 參考來源
本研究涵蓋的六個業界方案與標準
🔵

Google A2A Protocol v1.0

2025 年發布的 Agent-to-Agent 互操作標準。Agent Card 格式、Task 生命週期、SSE 推送規範。

🟠

AGNTCY — Agent Directory Service

Cisco 提出的 ADS（Agent Directory Standard）。DHT 分散式索引，適用完全去中心化場景。arxiv 論文。

🟣

Clawdentity Protocol（IETF Draft）

2026-02 IETF Draft。Ed25519 + AIT（Agent Identity Token）Pairing Ceremony，確立 Ed25519 為 Agent 身份業界標準。

🟡

Hyperspace AGI

libp2p + GossipSub + CRDT 實作的開放 P2P Agent 網路。Ed25519 keypair per-Agent，適合公共市場場景。

🟢

SPIFFE/SPIRE（CNCF）

雲原生 Workload Identity 標準。X.509 SVID 短期憑證 + 自動輪替 + mTLS，Production 級推薦方案。

⚡

NATS JetStream

Subject-per-Skill 路由模式。持久化消息隊列 + Dead Letter Queue + Circuit Breaker 集成，Phase 2 升級目標。
