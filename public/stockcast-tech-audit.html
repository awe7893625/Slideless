Section 06

IEObserve 轉錄完整流程

從 RSS 抓取到快取存檔的 7 步驟自動化管線

1

RSS 抓取

訂閱兩個 SoundOn RSS Feed：
IEO看什麼：feeds.soundon.fm/podcasts/c92cb66a-14fc-412f-84aa-08125add5ca9.xml
異溫層：另一個 feed URL
解析最新集數，取得音訊下載連結與集數元資料。

2

音訊下載

從 RSS 取得的 URL 下載 .mp3 / .m4a 音訊檔案，
暫存至 /tmp/ieobserve/ 目錄，完成後自動清除暫存。

3

Whisper 本機轉錄

使用開源 Whisper large-v3（最高品質模型）進行本機轉錄：
language=zh（繁體中文）、fp16=False（CPU 模式，不需 GPU）
預估轉錄時間：30–45 分鐘/集。

4

去重複後處理

呼叫 _dedupe_whisper_repeats() 函數，
移除 Whisper 常見的重複詞彙 Hallucination，
例：「你要你要你要這樣做」→「你要這樣做」。

5

Gemini 2.0 Flash 分析

將完整逐字稿送入 Gemini 2.0 Flash 進行語意分析：
輸出：3–5 個重點摘要 + 分類標籤
分類標籤：AI / 半導體 / 科技 / 總經

6

快取存檔

分析結果寫入 JSON 快取：
~/.claude/cache/ieobserve-podcast-latest.json
下次主報告執行時直接讀取，無需重新轉錄。

7

讀入主報告

generate_stockcast.py 於 08:00 / 20:00 執行時，
讀取上述 JSON 快取，注入 Podcast 分析區塊 HTML，
與即時市場數據合併後部署至 Vercel。

Section 07

校稿機制說明

目前的自動校正設計與已知限制

重要：系統目前無人工校稿步驟。從 Whisper 轉錄到 Gemini 分析到報告上線，全流程全自動執行，不包含人工審查環節。
自動校正機制（唯一）
_dedupe_whisper_repeats()
刪除 Whisper 重複詞彙 Hallucination
例：「這個這個這個方向」→「這個方向」
基於正則表達式比對，無 LLM 成本
MacroMicro Hallucination Filter
移除 LLM 常見注入詞
例：「请不吝点赞」「如果您喜欢」
基於關鍵字黑名單過濾
已知限制
限制項目 影響 建議改善
Gemini 分析不經人工審查 可能包含錯誤解讀或誤解語境 建立 confidence score 低於閾值時標注
Whisper 重大轉錄錯誤無法被捕捉 人名/股票代號可能被誤轉 加入財經術語詞彙表（vocabulary bias）
Capitol Trades HTML 結構依賴性 網站改版 → 解析失敗 → 空資料 加入欄位完整性驗證 + Discord 告警
