Section 1 — LLM 機制
大型語言模型的運作原理
T-TAX 以 Claude（Anthropic）為核心推理引擎，透過雙 Proxy 橋接訂閱制服務
🧠
LLM 基礎原理

LLM（大型語言模型）是一種經過海量文字資料訓練的神經網路，能夠理解並生成人類語言。模型在訓練過程中學習了語言模式、推理邏輯與領域知識，推理時透過「注意力機制（Attention）」決定哪些上下文對答案最重要。

T-TAX 使用 Claude（Anthropic） 作為核心語言推理引擎，負責法律判斷、選擇題推理與諮詢回答的最終生成。

🔌
雙 Proxy 橋接架構

T-TAX 不直接呼叫 Claude API，而是透過本地 Proxy 橋接：

主要路線： claw-proxy（port 38780）— 橋接 Claude Code 訂閱制，無 token 計費限制

備用路線： codex-proxy（port 38784）— 橋接 Codex CLI，主路線異常時自動切換

此架構讓 T-TAX 在訂閱制框架內運行，避免高頻查詢的 API 成本問題。

📋
LLM 接收的 Prompt 結構

T-TAX 的每次推理請求包含三層輸入：

System Prompt    → 角色設定（資深財稅顧問身份、語言規範、禁止條款）
Context Window  → 向量資料庫檢索出的 top-10 最相關法條片段
User Query       → 使用者的稅務問題（可能含補充事實）

────────────────────────────────────────────────────
LLM 不主動查詢法條，而是閱讀「預先檢索好的法條片段」做推理
法條由 TaxRAGService 從向量資料庫提取，再注入 prompt

1
TaxQueryRouter
問題分類（兩階段判斷）
優先用關鍵字規則比對（快速免費）判斷問題類型：稅種路由、計算題、一般查詢。MCQ 選擇題（含 (A)(B)(C)(D) 格式）偵測到後強制走推理路線，不走計算引擎。只有低信心時才呼叫 LLM 做語意分類，降低 API 成本。
兩階段判斷 關鍵字比對優先 MCQ 偵測 低信心才呼叫 LLM
2
TaxFactExtractor
事實提取與缺口分析
呼叫一次 LLM 分析問題是否缺少必要資訊（年度、身份別、適用對象、金額等）。輸出 missing_facts_count，數字越高代表問題越不完整，會降低最終答案的信心等級，並在輸出中提示使用者補充資訊。
呼叫 1 次 LLM missing_facts_count 信心等級影響
3
TaxRAGService
向量檢索 + 混合排序
LightVectorStore 向量資料庫儲存 2,443 份法規文件（1,411 條法律 + 921 條施行細則 + 111 條解釋令/函釋）。 Hybrid Search 同時執行向量相似度（OpenAI Embedding）與 BM25 關鍵字搜尋，合併取 top-20。 Rule Engine 針對特定稅種強制加入相關法條（確保不漏）。 最後 Reranker 精選 top-10 最相關法條片段，注入 LLM prompt。
Hybrid Search 2,443 份文件 向量 + BM25 Reranker top-10 Rule Engine
4
Reasoner Layer
法律推理 / 計算引擎 / 申報程序
TaxLegalReasoner（主路線）：Claude 語言模型根據檢索到的法條進行法律推理，適用選擇題與法律判斷問題。
TaxCalculationReasoner：純 Python 計算引擎，處理計算題不呼叫 LLM，確保數值精準。
TaxFilingReasoner：處理申報程序、截止日期等程序性問題。
知識圖譜（KG）驗證：交叉確認 LLM 引用的法條是否真實存在於資料庫中，防止幻覺引用。
Exam Mode：放寬法條限制，允許 LLM 運用專業知識補充，適用考試情境。
Claude 主力推理 純 Python 計算 KG 幻覺防護 Exam Mode

Section 4 — 計算引擎
計算題的處理方式
純 Python 數值計算，不呼叫 LLM，確保稅額計算的精準性

計算題偵測邏輯

  • Router 偵測到純計算意圖關鍵字：「計算」「應納稅額」「試計算」等
  • 且非選擇題格式（選擇題會強制走推理路線）
  • 轉派 TaxCalculatorService（Python 計算引擎）
  • 計算引擎直接輸出數值結果，不經 LLM 推理
稅種 計算邏輯 稅率結構
個人綜合所得稅 10 類所得加總 → 減免扣除項目 → 適用累進稅率 5% / 12% / 20% / 30% / 40%（五級累進）
營利事業所得稅 課稅所得 × 稅率（12萬以下免稅） 20%（單一稅率）
遺產稅 遺產淨額（扣除免稅額與扣除額）× 累進稅率 10% / 15% / 20%（三級累進）
贈與稅 贈與淨額 × 稅率 10%（單一稅率）
最低稅負（AMT） 基本所得額超過 670 萬元的部分 × 稅率 20%（超額部分）
Section 5 — Agent 設定
TaxLegalReasoner 人格設定與規範
T-TAX Agent 的系統提示（System Prompt）設計，決定 AI 的行為邊界與輸出品質
👤
AI 身份設定

角色：資深財稅顧問，熟稔台灣稅法實務

語言：繁體中文，語氣專業但易懂

一般模式限制：只能使用 RAG 提供的法條清單作答，不得引用清單以外的法條（防止幻覺）

考試模式限制：優先使用提供的法條，若法條不足可運用專業知識補充（放寬限制以提升答題率）

硬性禁止：提供「確定的法律建議」，輸出必須包含免責聲明

Section 6 — Architecture Summary
系統架構全景
T-TAX 的技術棧與各元件的職責分工

設計理念：兩階段判斷 × 成本最優化

  • Router 與 Extractor 優先用規則比對（免費、快速），只有低信心時才呼叫 LLM
  • RAG 與 KG 驗證完全不呼叫 LLM，使用純向量計算與圖查詢
  • 計算題使用 Python 引擎而非 LLM，確保數值精準且零 AI 誤差
  • 主要 LLM 呼叫集中在最終推理（Layer 4），每次查詢通常只呼叫 1–2 次 Claude
  • 雙 Proxy 架構確保服務連續性，claw-proxy 主路線異常時自動切換 codex-proxy
