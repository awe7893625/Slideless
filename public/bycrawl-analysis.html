02 · API 結構完整解析
URL Pattern + Auth + Credit 系統

Base URL：https://api.bycrawl.com
認證：Header x-api-key: sk_byc_xxx
格式：純 JSON，無外層 wrapper object（直接回傳 data）

統一 URL Pattern
GET /{platform}/users/{username}         # 用戶資料
GET /{platform}/users/{id}/posts         # 用戶貼文
GET /{platform}/posts/{id}               # 單篇貼文
GET /{platform}/posts/search?q=keyword   # 搜尋
GET /{platform}/feed/public              # 公開 Feed
GET /{platform}/bulk/{jobId}             # 批次操作

分頁機制
使用 cursor 參數做 cursor-based 分頁。回應包含 pageInfo.endCursor 或 nextCursor，帶入下一次請求即可取得下一頁。Python SDK 的 iter_*() 方法自動處理分頁。

05 · Python SDK 設計分析
官方 SDK 的優秀設計值得借鑑
from bycrawl import ByCrawl

client = ByCrawl()  # 讀 BYCRAWL_API_KEY 環境變數

# 統一 pattern：client.{platform}.{action}()
user = client.threads.get_user("zuck")
print(user.data.username)       # 型別化 autocomplete
print(user.credit.remaining)    # 剩餘 credits

# 自動分頁 iterator
for post in client.threads.iter_user_posts("zuck"):
    print(post.text)

# 跨平台一致
jobs = client.linkedin.search_jobs("AI engineer", location="Taiwan")
posts = client.reddit.get_subreddit_posts("taiwan", sort="hot")
tweets = client.x.get_user_posts("elonmusk", count=40)
SDK 亮點
✓ 型別化回應（APIResponse[T]）
✓ iter_*() 自動分頁
✓ Async 支援（AsyncByCrawl）
✓ 自動 retry + exponential backoff
✓ credit / rate limit metadata
✓ BYCRAWL_API_KEY 環境變數

08 · 實作路線圖
三個 Phase 逐步交付
Phase 1 — 閘道 + Reddit + YouTube（1 週）
Rex 建立 FastAPI 閘道層（API Key 認證 + Credit 計數）
Spider 實作 Reddit adapter（公開 JSON API）
Spider 實作 YouTube adapter（Data API v3）
→ 驗證整個 pipeline 跑通後進 Phase 2
Phase 2 — Playwright 系平台（2–3 週）
Nautilus 實作 Threads / Instagram adapter
Nautilus 實作 Twitter/X adapter（繞過反爬）
Nautilus 實作 LinkedIn adapter
Nautilus 實作 TikTok adapter（需要 headless + 等待 20–40s）
→ 每個平台完成後獨立測試驗收
Phase 3 — Python SDK + MCP（1 週）
Rex 發布 pip install rain-crawl Python SDK
複製 byCrawl SDK 的 API 設計（client.threads.get_user()）
Max 建立 MCP 套件，讓 AI agents 可直接呼叫
→ 對接 GooayeAI / EchoMarketer / Buzz 使用

⚠️ 重要風險提示：Threads/Instagram/Facebook（Meta 系）、LinkedIn 有嚴格反爬機制，Playwright 方案需要持續維護（帳號池、IP 輪換、反指紋）。Twitter/X 同樣有封禁風險。建議 Phase 2 先做 Dcard/Job104 確認 pipeline 正確，再攻 Meta 系。
