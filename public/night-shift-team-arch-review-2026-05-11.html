Findings

架構與角色配置問題

Roster inflation

56 個 active 非工具/非 frozen 角色中，本週只有 8 個非 Kai agent 有 completion。Investment 與 external worker 名單比任務證據大很多。

Night assignment drift

夜班協議明確不派 Ray，但本任務在 Kai 仍是 assigned_to=`ray`。這會污染 Ray throughput 與責任歸屬。

Generic task_type

`analysis`、`feature` 被多個角色共用，導致 Quill/Forge/Ray 這類 gate role 也收到像 implementation 的 task_type。

Recurring backlog

Orion 的 `Module R1` 與 `知識管線升級` 跨日重複 pending；應有 WIP cap 與 duplicate suppression。

External lane overlap

Spider/Lynx/Nautilus/Probe/Echo 與 Beaver Badger/Otter/Shutter/Ferret 的邊界需要更硬的 dispatch criteria。

Metric endpoint drift

`/api/metrics/agent-performance` 本週回空，因為 endpoint 查小寫 status，但 DB 是大寫 enum。

Recommendations

具體改善建議

  1. 在 `team.json` 增加 `active_execution / on_demand / reserve / tool / frozen` 分層，weekly KPI 只計 active_execution。
  2. 移除 `Milo`、`Sage` 在部門與 frozen list 的重複記錄，只保留單一 canonical frozen entry。
  3. 修復 `/api/metrics/agent-performance` 的 status enum 查詢，改用 `UPPER(status)` 或 ORM enum。
  4. 夜班建立任務時把 Ray 放在 reviewer/governance_owner，不放 assigned_to，除非真的派 Ray 執行。
  5. 把 Klay 的非決策型 backlog 改派給 Forge/Webb/Orion/Quill/Max/Rex/John/Iris 等實際 owner。
  6. 對 Orion recurring modules 加 WIP cap；前一天仍 pending 時不可繼續建立下一個同類任務。
  7. Noel、Silo 在 open failed/executing 任務清掉前標記 degraded，避免自動派新工作。
  8. 把 Kai aggregate `beaver` evidence 映射到 named Beaver roles，否則 named roles 應列 execution pool 而不是 active agent。
  9. 啟用 Operations Manager 管 weekly backlog triage，啟用 Logger 管 digest/writeback，降低 Klay/Orion 壓力。
  10. Investment Phase B+ analysts 暫列 on-demand/reserve，等 active research cron 跑通再升回 active execution。
