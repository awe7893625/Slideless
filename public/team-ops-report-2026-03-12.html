🔒 安全強化

三層防禦機制

dispatch 函數硬化 + terminal 指令白名單 + shell injection 修復。Quill 審查發現的 2 個漏洞也已修補。

HIGH
run_terminal_command 白名單 + 黑名單
system_tools.py — 39 個允許指令 + 15 個危險模式
↓
白名單：只允許 39 個指令前綴（ls, git, curl, python3, open, tmux, openclaw 等）。管道命令拆解後逐一檢查。

黑名單：15 個危險 regex 模式，包含 rm -rf /、mkfs、dd if=、fork bomb、curl|bash、DROP TABLE、eval $()、反引號命令替換、$() 命令替換。

Quill W2 修復：黑名單新增 `...` 和 $( 模式，防止反引號/命令替換繞過白名單。
HIGH
open_application Shell Injection 修復
system_tools.py — Quill W3 漏洞修補
↓
問題：open_application() 用 f-string 拼接 shell 指令（open -a '{app_name}'），單引號可被注入。

修復：改用 asyncio.create_subprocess_exec("open", "-a", app_name)，完全繞開 shell 層，消除注入風險。
MID
dispatch_tools 全面硬化
4 項改善：exit code / JSON 驗證 / agent 白名單 / task_id 驗證
↓
_run_spawn：檢查 exit code（非零附加 exit_code 欄位），驗證輸出是否為合法 JSON（非法則回傳前 200 字供 debug）。

dispatch_openclaw：ALLOWED_OPENCLAW_AGENTS 白名單（spider/lynx/nautilus/probe/echo/noel/buzz），拒絕未知 agent。

complete_task：task_id 用 re.fullmatch(r"[A-Za-z0-9_\-]{1,64}") 驗證。
MID
tmux return code + 收尾保障
spawn-klay/worker/ray.py — 防看板孤兒任務
↓
tmux check：三支 spawn 腳本在 proc.wait() 後檢查 return code，非零則立即 fail_kai_task。

Shell fallback：Claude/Codex 退出後，shell 腳本自動檢查任務狀態。若仍在 executing 或 pending，根據 exit code 自動 /done 或 /fail。徹底解決「看板執行中顯示已結束任務」。
✓ DONE
dispatch_openclaw 看板登記
OpenClaw 任務自動建立 Kai 任務追蹤
↓
問題：spawn-klay/worker/ray 自動登記看板，但 dispatch_openclaw 完全不經 Kai，無法追蹤。

修復：新增 _kai_create_task()、_kai_complete_task()、_kai_fail_task() 三個 helper。Best-effort 設計，看板 API 失敗不阻擋主流程。
