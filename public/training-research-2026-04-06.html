Key Finding: 轉換邏輯

Board Decision 是 single source of truth，所有訓練內容按 verdict 欄位分級：

  • approve — 全員訓練（必讀，納入週訓練更新）
  • design only — 認知層（了解概念，不需實作）
  • spike only — 選讀（有興趣者深入研究）
  • monitored — 不進訓練（持續觀察，暫不行動）

業界自我訓練模式

Reflexion
語言強化學習

Agent 用自然語言反思失敗原因，存入「經驗記憶」。下次嘗試時注入反思文字作為額外 context。GPT-4 在 HumanEval 上從 80% 提升至 91%，僅靠語言反思而非參數更新。

SICA
自我改進程式碼 Agent

Agent 修改自身的程式碼（工具定義、策略函數）。在 SWE-Bench 上從 17% 提升至 53%。關鍵：Agent 不只改產出物，還改自己的推理流程。

Voyager
技能庫模式

Minecraft 裡的終身學習 Agent。自動課程（自動設定目標）+ 技能庫（成功的程式碼段存入可複用庫）+ 技能檢索（新任務時找最相關的舊技能）。我們的 skills/ 目錄就是簡化版 Skill Library。

EvoSkill
從失敗軌跡自動發現技能

分析大量任務執行軌跡，自動識別重複出現的成功模式，將其抽象為「可演化的技能」。技能本身也會隨時間演化、合併、淘汰。最接近「AI 自動寫 SOP」的框架。

Section 03

我們已做對的 vs 可升級的

Rain AI Team 現有機制與業界最佳實踐的對照，找出差距與升級點。

● 已做對

Session Review 五問 = Reflexion 人工版

每次 session 結束強制反思「遇到什麼問題、如何解決、下次如何改善」，本質上就是 Reflexion 的語言化反思迴圈。

lessons.md FTS5 = 簡易 RAG

錯誤記錄寫入 lessons.md，下次任務前用 FTS5 全文搜尋檢索相關教訓。等同 Reflexion 的「經驗記憶檢索」。

Scope Card + 3-Phase = 品質閘門

R-00 強制 Scope Card + Phase 1/2/3 規劃協議，確保任務經過商業/架構/風險三層審查才進入實作。

Error Registry E001-E008 = 結構化失敗分類

8 種錯誤代碼（未查證推測、自行降標、假完成...），累積 3 次自動升級為 rule/skill/hook。這是 EvoSkill 模式的人工版。

● 可升級

自動化 Reflexion 迴圈

目前反思靠 session review 手動觸發。升級為結構化 JSON 反思 API，每次任務結束自動提交、自動檢索。

技能演化管線（EvoSkill 模式）

目前 skills/ 手動新增。可分析任務軌跡，自動發現重複成功模式，提案新 skill 或合併/淘汰舊 skill。

Agent 效能儀表板

per-agent metrics（成功率、平均耗時、token 效率、常見錯誤類型）。目前只有粗顆粒度的 task board 資料。

Prompt 自動優化（DSPy）

Stanford DSPy 框架可自動優化 prompt，用少量標註資料找到更好的 prompt 寫法。可用於 Agent system prompt 調優。
