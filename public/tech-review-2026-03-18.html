第一次 No-Go — 安全風險過高

🚫 No-Go（Veto 生效）

完整 P2P 節點模式存在 CRITICAL 等級安全風險，Ray 行使 veto 否決整合：

風險項目 等級 說明
WASM 外部代碼執行 CRITICAL 匿名節點推來的任意邏輯在本機執行
P2P 網路暴露 HIGH 本機直接連接全球不可信節點
pipe-to-bash 安裝 HIGH curl | bash 安裝，無代碼審查
localhost:8080 無認證 MEDIUM 本地 API 無任何身份驗證
深研發現新事實：存在 --api 模式（只跑本地推論，不加入 P2P）。CRITICAL 和 HIGH 風險全部消除。→ 重新送審
🔴
Ray · 第二次審查 · Skeptic Check 命中根本問題

第二次 No-Go — 功能重複，已有更好替代

🚫 No-Go（Skeptic Check）

Ray 的 Skeptic Check 提出關鍵問題，命中根本問題：

「API-only 模式 = 功能更少的 Ollama。
Ollama 已安裝在系統上（v0.17.4），OpenAI-compatible API，零額外風險，無需引入任何閉源依賴。
如果根本不裝 Hyperspace，用 Ollama 就好，為什麼要引入更多依賴？」

最終決定：不整合 Hyperspace。改用已安裝的 Ollama 做本地推論 fallback。

✅ 最終決策：使用 Ollama v0.17.4（已安裝）作為本地推論 fallback，OpenAI-compatible API，零額外風險，無任何新依賴。

Section 4 · Chrome DevTools MCP

Chrome DevTools MCP — 安裝完成

接管 Rain 的真實 Chrome Session · 含 Google、GA4、GitHub 後台
狀態：✅ 已安裝，✓ Connected
chrome-devtools: npx chrome-devtools-mcp@latest --autoConnect - ✓ Connected
這是什麼？

不是開新無頭瀏覽器，是接管 Rain 正在用的 Chrome（你的 Google、GA4、GitHub 後台都在裡面）。50+ 工具：click、fill、navigate、screenshot、console log、network 分析、Core Web Vitals。

使用方式（每次要用前）
# 步驟 1：用偵錯模式重開 Chrome
bash ~/.claude/scripts/chrome-debug.sh

# 步驟 2：重啟 Claude Code session
# 之後就可以說：「幫我看 GA4 昨天流量最高的文章」
Step-by-Step 啟動流程
  • 1
    執行啟動腳本 bash ~/.claude/scripts/chrome-debug.sh — 以偵錯模式重啟 Chrome
  • 2
    重啟 Claude Code 關閉並重開 Claude Code session，讓 MCP server 重新連接
  • 3
    直接描述需求 「幫我查 GA4 昨天流量」/ 「幫 Webb 截圖 TheOrder 首頁」
  • 4
    付款/刪除操作 → Rain 確認 AI 停在確認頁前，Rain 審查後才繼續（見下方安全護欄）
實際使用場景
  • 幫 Atlas 查 Google Analytics 流量，不需要 OAuth token
  • Webb 做網站 QA，截圖 + 看 console 錯誤，確認 Core Web Vitals
  • 讀 GitHub 後台，整理 starred repos 或 PR 狀態
  • 查看 TheOrder 後台商品狀態（但付款/刪除操作需 Rain 確認）
  • 協助 Atlas 做 GA4 報表分析（無需額外 token）
安全護欄（已加進 exclusion-rules.md C-1～C-5）
