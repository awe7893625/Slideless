05 — 逆向發現

FAP 的 API 直發技術

最強大的技術路線 — 繞過 DOM 直接呼叫 Facebook 內部 GraphQL

GraphQL 直發

不操作 DOM，直接呼叫內部 API

FAP 逆向了 Facebook 的 GraphQL endpoint，用 ComposerStoryCreateMutation 直接建立貼文，完全不需要開啟社團頁面。

Header 偽造

declarativeNetRequest 偽造 Origin/Referer

使用 Chrome MV3 的 declarativeNetRequest API 在網路層級修改 request header，讓 Facebook 認為請求來自合法頁面。

Token 提取

fb_dtsg + c_user 動態取得

從已登入的 Facebook 頁面中提取 fb_dtsg CSRF token 和 c_user cookie，拼成合法的 API 請求參數。

圖片上傳

upload.facebook.com 獨立通道

圖片透過 upload.facebook.com 專用端點上傳，取得 photo_id 後附加到貼文 mutation 中。

Phase 2 參考價值：此技術可實現圖片發文，但風險高（API 變動、帳號封鎖）。我們將在 Phase 2 評估是否採用混合方案。

06 — 逆向發現

人類模擬技術

kevindemara 的高擬真操作模擬 — 讓自動化行為看起來像人類

四大模擬技術
  • moveCursor() — 10 步滑鼠軌跡模擬
    不是瞬間移動到目標，而是分 10 步漸進移動，模擬人手抖動
  • window blur/focus — 視窗切換模擬
    隨機觸發 blur/focus 事件，模擬使用者在多個分頁間切換
  • execCommand("insertHTML") — React 相容文字插入
    繞過 React 虛擬 DOM 直接注入文字，比 value 賦值更穩定
  • 分層隨機延遲 — 四層延遲機制
    初始延遲 / 打字間隔 / 點擊間隔 / 關閉等待，每層獨立隨機
模擬層典型延遲隨機範圍
初始等待2-5 秒±50%
打字間隔50-150ms/字±30%
點擊間隔300-800ms±40%
關閉等待1-3 秒±50%

我們的取捨：mbasic 路線天生低風險（行為簡單），不需要完整的人類模擬。但智慧延遲已採納隨機抖動概念。

09 — 新功能深入

Spintax 引擎

來自 Group Posting PRO 的核心防重複技術

語法示範
// 基本語法 {你好|哈囉|嗨}，我想分享一個{好用|實用|方便}的工具 // 巢狀語法 {{早安|午安}各位|大家{好|嗨}} // 實際輸出範例 社團 A → "哈囉，我想分享一個方便的工具" 社團 B → "嗨，我想分享一個好用的工具" 社團 C → "你好，我想分享一個實用的工具"
技術規格
  • 遞迴解析，支援無限巢狀
  • 每次執行獨立隨機種子
  • 空選項支援：{有文字|}
  • 不影響非 Spintax 文字
防重複效果
  • 3 個變體 × 3 個位置 = 27 種組合
  • 搭配 3-Slot 輪替 = 81 種組合
  • Facebook 重複內容偵測幾乎無法觸發

11 — 新功能深入

3-Slot 內容輪替 + 即時進度

源自 FAP 的多組文案策略 + 團隊原創的進度追蹤

3-Slot Round-Robin 輪替
// 三組文案自動輪替 Slot A: "限時優惠！{立即|馬上}搶購" Slot B: "好物分享 — {不買可惜|錯過可惜}" Slot C: "社團{專屬|限定}優惠來了" // 發文順序 社團 1 → Slot A（Spintax 變體 1） 社團 2 → Slot B（Spintax 變體 2） 社團 3 → Slot C（Spintax 變體 3） 社團 4 → Slot A（Spintax 變體 4） // 循環
組合威力
  • 3 Slot × Spintax 變體 = 每篇文都不同
  • 即使 Facebook 比對相鄰貼文也找不到重複
  • Slot 2、Slot 3 為選填，最少用 1 組也行
即時進度 UI
  • ✓ 成功計數（即時更新）
  • ✗ 失敗計數 + 失敗社團名稱
  • ⏱ 下一篇倒數計時
  • ⚡ 完成摘要報告（成功率 + 耗時）

14 — 風險評估

已知風險 & 緩解策略

四大風險項目，目前整體風險可控

R1 — mbasic.facebook.com 被關閉 中機率

影響：整個發文功能失效。緩解：Phase 2 準備 m.facebook.com 備援路線；API 直發作為第三選項。mbasic 已存在 15+ 年，短期關閉機率低。

R2 — Spintax 模式被 Facebook 識破 低機率

影響：重複內容偵測觸發。緩解：3-Slot 輪替讓內容差異更大；使用者可自行增加 Spintax 變體數量。

R3 — Phase 2 圖片方案失敗 中機率

影響：無法支援圖片發文。緩解：準備三條路線（m.fb / API / 純文字 fallback），至少一條會成功。

R4 — Facebook 加強反自動化偵測 中機率

影響：帳號限制或暫停。緩解：智慧延遲 + Spintax + 內容輪替已覆蓋 5/7 防封機制；使用者控制發文頻率。
