P2P 穿透原理（為什麼不一定需要 Relay）

# RustDesk 連線流程

Step 1: Client A 啟動 → 向 hbbs 註冊 ID（UDP 21116）
Step 2: Client B 要連 A → 向 hbbs 查詢 A 的 public IP:port
Step 3: hbbs 通知雙方互相 UDP punch
Step 4: P2P 穿透成功 → 直連（延遲 <50ms，零中繼流量）
Step 4b: P2P 失敗（對稱 NAT）→ 走 hbbr relay（延遲 +30-80ms）

穿透成功率：
  Cone NAT（家用路由器）    → ~85% 成功
  對稱 NAT（企業防火牆）   → ~30% 成功，其餘走 relay
  全域 relay fallback       → 100% 連線率（一定能連）
Infra 開工前提：Rain 確認用 Mac Docker 還是開 VPS。不影響其他模組開發 — AI Agent 層用 localhost 測試即可。
02 — AI Control Implementation

AI 三模式控制（實作細節）

每種模式的 code 層級設計。控制通道統一走 Agent Daemon WebSocket（Section 10），不依賴 RustDesk SSH tunnel。RustDesk 僅負責 P2P 連線建立和畫面串流，AI 控制指令全走 Daemon。

架構決策（v2.0 明確）：RustDesk = 連線層（P2P 穿透 + 畫面串流 + 客戶即時觀看）。AI 控制指令 = Daemon WebSocket（Kai Server ↔ Client Daemon）。兩條通道獨立運作，RustDesk 宕機不影響 AI 指令執行，Daemon 斷線不影響客戶觀看畫面。

# --- macOS ---
class MacSystemAgent:
    def install_app(self, app_name):
        """用 Homebrew 安裝"""
        return self.ssh_exec(f"brew install {app_name}")

    def open_app(self, app_name):
        """用 AppleScript 開啟 App"""
        return self.ssh_exec(
            f'osascript -e "tell application \\"{app_name}\\" to activate"'
        )

    def click_ui(self, x, y):
        """AppleScript 模擬點擊"""
        script = f'tell application "System Events" to click at {{{x}, {y}}}'
        return self.ssh_exec(f'osascript -e "{script}"')

    def type_text(self, text):
        """AppleScript 模擬鍵盤輸入"""
        script = f'tell application "System Events" to keystroke "{text}"'
        return self.ssh_exec(f'osascript -e "{script}"')

# --- Windows ---
class WinSystemAgent:
    def install_app(self, app_name):
        return self.ssh_exec(f"winget install {app_name}")

    def run_powershell(self, script):
        return self.ssh_exec(f"powershell -Command '{script}'")

# --- 統一介面 ---
class SystemAgent:
    def __init__(self, os_type):
        self.agent = MacSystemAgent() if os_type == "macos" else WinSystemAgent()

# Mode C: Vision Agent 實作

import anthropic

class VisionAgent:
    def step(self, instruction: str):
        # 1. 截圖
        screenshot = self.take_screenshot()  # RustDesk 截圖 API

        # 2. 送給 Claude Vision 分析
        response = anthropic.messages.create(
            model="claude-sonnet-4-6",
            messages=[{
                "role": "user",
                "content": [
                    {"type": "image", "source": screenshot},
                    {"type": "text", "text": f"""
                        你正在看一台遠端電腦的螢幕截圖。
                        目標：{instruction}
                        請回傳：要點擊的座標 (x, y) 或要輸入的文字。
                        格式：{{"action": "click", "x": 500, "y": 300}}
                    """}
                ]
            }]
        )

        # 3. 執行動作
        action = parse_json(response)
        if action["action"] == "click":
            self.mouse_click(action["x"], action["y"])
        elif action["action"] == "type":
            self.keyboard_type(action["text"])

        # 4. 截圖驗證
        verify = self.take_screenshot()
        return self.verify_success(verify, instruction)

# RemoteController — 自動選擇最佳控制模式

class RemoteController:
    def execute(self, task):
        # 1. 判斷任務類型 → 選模式
        if task.type == "browser":
            result = self.browser_agent.execute(task)       # Mode A
            if result.failed:
                result = self.vision_agent.execute(task)    # Fallback → C

        elif task.type == "install" or task.type == "system":
            result = self.system_agent.execute(task)       # Mode B
            if result.failed:
                result = self.vision_agent.execute(task)    # Fallback → C

        elif task.type == "unknown":
            result = self.vision_agent.execute(task)       # Mode C 直接

        # 2. 全部失敗 → 升級人工（TeamViewer）
        if result.failed:
            self.escalate_to_human(task)                   # TeamViewer

        return result

Install Token 機制（安全核心）

# Token 生命週期

Console（你）→ 產生 Token → 烤入 Installer → 給客戶
                                                    ↓
                                              客戶雙擊安裝
                                                    ↓
                                           Daemon 用 Token 註冊
                                                    ↓
                                       Server 驗證 Token → 核發 device_id
                                                    ↓
                                           Token 自動失效（一次性）
屬性規格
格式tok_ + 32 位 hex（tok_a3f2c8e1b9d4...）
生成方式Console UI → POST /api/device/token → 綁定客戶名稱
有效期72 小時（過期自動失效）
使用次數一次性（註冊成功後立即標記 used）
烤入方式打包 Installer 時寫入 config.json
無 Token 安裝Daemon 啟動 → 註冊失敗 → 不連線（安全）

                       │                    │      │

# 狀態說明
IDLE        → 等待連線
CONNECTING  → SSH tunnel 建立中 + RustDesk handshake
CONNECTED   → 已連線，等待指令
EXECUTING   → 正在執行指令
FALLBACK    → 當前模式失敗，切換備援模式
RETRY       → 連線/指令失敗，重試中
FAILED      → 重試用盡，需人工介入
ESCALATED   → 已升級到 TeamViewer 人工操作
CLOSED      → Session 正常結束

# 自動過期規則
CONNECTED 閒置 30 分鐘 → 自動 CLOSED
EXECUTING 超時 → 自動截圖 + 標記 timeout
FAILED 超過 1 小時無人處理 → Discord 再次通知

Daemon 主程式架構

# scripts/remote_agent_daemon.py
# 客戶端常駐程序 — 單一 Python 檔案，無外部依賴（除 websockets）

import asyncio, json, platform, subprocess, os, time, logging
import websockets
from pathlib import Path

# ── 設定 ──
KAI_SERVER   = os.getenv("KAI_SERVER", "ws://YOUR_SERVER:5002")
DEVICE_ID    = os.getenv("DEVICE_ID", "")  # 安裝時寫入
AUTH_TOKEN   = os.getenv("AUTH_TOKEN", "") # 安裝時寫入
HEARTBEAT_INTERVAL = 10   # 秒
RECONNECT_BACKOFF = [1, 3, 10, 30, 60]  # 秒
LOG_DIR      = Path("/opt/rain-remote") / "logs"
STATE_FILE   = Path("/opt/rain-remote") / "state.json"

log = logging.getLogger("rain-remote")

class AgentDaemon:
    def __init__(self):
        self.ws = None
        self.session_id = None
        self.capabilities = self.detect_capabilities()
        self.command_handlers = {
            "browser.navigate":  self.handle_browser_navigate,
            "browser.click":     self.handle_browser_click,
            "browser.fill":      self.handle_browser_fill,
            "browser.screenshot": self.handle_browser_screenshot,
            "browser.evaluate":  self.handle_browser_evaluate,
            "system.shell":      self.handle_system_shell,
            "system.install":    self.handle_system_install,
            "system.file_write": self.handle_file_write,
            "system.file_read":  self.handle_file_read,
            "vision.analyze":    self.handle_vision_analyze,
            "vision.click_at":   self.handle_vision_click,
            "meta.heartbeat":    self.handle_heartbeat,
            "meta.capabilities": self.handle_capabilities,
        }
        self.chrome_process = None  # Chrome CDP 程序追蹤
        self.seq = 0

    def detect_capabilities(self):
        """偵測這台電腦能做什麼"""
        os_type = platform.system().lower()  # darwin/windows/linux
        caps = {
            "os": os_type,
            "os_version": platform.version(),
            "hostname": platform.node(),
            "python": platform.python_version(),
            "chrome": self._check_chrome(),
            "brew": os_type == "darwin" and self._cmd_exists("brew"),
            "winget": os_type == "windows" and self._cmd_exists("winget"),
            "ssh": self._cmd_exists("ssh"),
            "node": self._cmd_exists("node"),
            "git": self._cmd_exists("git"),
            "modes": []  # 可用模式清單
        }
        if caps["chrome"]: caps["modes"].append("browser")
        caps["modes"].append("system")   # 永遠可用
        caps["modes"].append("vision")   # 永遠可用（截圖就行）
        return caps

    async def run(self):
        """主迴圈：連線 → 心跳 → 收指令"""
        retry_idx = 0
        while True:
            try:
                url = f"{KAI_SERVER}/ws/remote/{DEVICE_ID}"
                headers = {"Authorization": f"Bearer {AUTH_TOKEN}"}
                async with websockets.connect(url, extra_headers=headers) as ws:
                    self.ws = ws
                    retry_idx = 0  # 連線成功，重置退避
                    log.info(f"Connected to {url}")

                    # 1. 註冊能力
                    await self.send_capabilities()

                    # 2. 並行：心跳 + 收指令
                    await asyncio.gather(
                        self.heartbeat_loop(),
                        self.command_loop()
                    )
            except (websockets.ConnectionClosed, ConnectionRefusedError, OSError) as e:
                wait = RECONNECT_BACKOFF[min(retry_idx, len(RECONNECT_BACKOFF)-1)]
                log.warning(f"Disconnected: {e}. Retry in {wait}s")
                await asyncio.sleep(wait)
                retry_idx += 1

    async def command_loop(self):
        """接收並執行指令"""
        async for raw in self.ws:
            msg = json.loads(raw)
            if msg["type"] != "command": continue
            cmd = msg["payload"]
            handler = self.command_handlers.get(cmd["action"])
            if not handler:
                await self.send_response(cmd["command_id"], "failed",
                    error=f"Unknown action: {cmd['action']}")
                continue
            # 非同步執行（不阻塞心跳）
            asyncio.create_task(self.execute_with_timeout(handler, cmd))

    async def execute_with_timeout(self, handler, cmd):
        """帶超時的指令執行"""
        timeout = cmd.get("timeout_ms", 30000) / 1000
        try:
            result = await asyncio.wait_for(handler(cmd), timeout=timeout)
            await self.send_response(cmd["command_id"], "success", result=result)
        except asyncio.TimeoutError:
            screenshot = await self.take_screenshot()  # 超時截圖保存現場
            await self.send_response(cmd["command_id"], "timeout",
                error=f"Timeout after {timeout}s", screenshot=screenshot)
        except Exception as e:
            await self.send_response(cmd["command_id"], "failed", error=str(e))

# backend/app/services/remote_decision_engine.py

from enum import Enum

class TaskType(Enum):
    BROWSER = "browser"
    SYSTEM  = "system"
    VISION  = "vision"
    UNKNOWN = "unknown"

class ControlMode(Enum):
    BROWSER_CDP = "browser"   # Mode A
    SYSTEM_CMD  = "system"    # Mode B
    VISION_AI   = "vision"    # Mode C
    ESCALATE    = "escalate"  # 人工

BROWSER_KW = {"登入","網站","部署","saas","後台","cms","表單",
    "login","deploy","dashboard","upload","vercel","github"}
SYSTEM_KW = {"安裝","設定","config","brew","npm","git","install",
    "環境","env","啟動","restart","terminal","檔案"}

class DecisionEngine:
    def classify_task(self, description: str) -> TaskType:
        """兩階段分類：關鍵字優先，LLM 備援"""
        desc_lower = description.lower()
        if any(kw in desc_lower for kw in BROWSER_KW):
            return TaskType.BROWSER
        if any(kw in desc_lower for kw in SYSTEM_KW):
            return TaskType.SYSTEM
        # Fallback: Haiku 分類（便宜 + 快）
        return self.llm_classify(description)

    def select_mode(self, task_type: TaskType, caps: dict) -> list[ControlMode]:
        """回傳 [主模式, fallback1, fallback2]"""
        if task_type == TaskType.BROWSER:
            if "browser" in caps["modes"]:
                return [ControlMode.BROWSER_CDP, ControlMode.VISION_AI, ControlMode.ESCALATE]
            return [ControlMode.VISION_AI, ControlMode.ESCALATE]

        if task_type == TaskType.SYSTEM:
            return [ControlMode.SYSTEM_CMD, ControlMode.VISION_AI, ControlMode.ESCALATE]

        # UNKNOWN / VISION → 直接 Vision
        return [ControlMode.VISION_AI, ControlMode.ESCALATE]

    async def execute_with_fallback(self, task, caps, agents):
        """依序嘗試模式鏈，直到成功或全失敗"""
        modes = self.select_mode(self.classify_task(task.description), caps)
        for mode in modes:
            if mode == ControlMode.ESCALATE:
                return await agents.escalate(task)
            result = await agents[mode].execute_with_retry(task, max_retries=3)
            if result.success:
                return result
            log.warning(f"{mode.value} failed, trying next")
        return await agents.escalate(task)

Agent Loop 程式碼

# backend/app/services/remote_agent_loop.py

from dataclasses import dataclass, field
from typing import Optional
from app.models.remote_models import (
    ActionType, CommandStatus, SessionStatus, ErrorCode
)

MAX_STEPS_PER_TASK = 50
MAX_VERIFY_RETRIES = 2
SCREENSHOT_DELAY_MS = 1000

@dataclass
class Observation:
    screenshot_b64: Optional[str]
    page_url: Optional[str]
    page_title: Optional[str]
    device_state: dict
    step_history: list[dict]
    remaining_steps: list[str]

@dataclass
class Decision:
    next_action: ActionType
    params: dict
    reasoning: str
    confidence: float
    needs_confirmation: bool = False

@dataclass
class VerifyResult:
    passed: bool
    evidence: str            # "URL changed to X" / "exit_code=0"
    before_screenshot: str
    after_screenshot: str

class AgentLoop:
    """AI Agent 的核心執行循環。"""

    def __init__(self, session, decision_engine, command_sender, planner, verifier):
        self.session = session
        self.engine = decision_engine
        self.sender = command_sender       # WebSocket command sender
        self.planner = planner             # LLM planner (Haiku/Sonnet)
        self.verifier = verifier           # 截圖比對 + 結果驗證
        self.step_count = 0
        self.step_history: list[dict] = []

    async def run_task(self, task_description: str) -> dict:
        """執行一個完整任務，回傳執行報告。"""

        # 拆解任務為步驟（LLM 規劃）
        plan = await self.planner.decompose_task(task_description)
        remaining = plan.steps[:]  # ["login", "deploy", "set_domain", "verify"]

        while remaining and self.step_count < MAX_STEPS_PER_TASK:

            # ═══ OBSERVE ═══
            obs = await self.observe(remaining)

            # ═══ DECIDE ═══
            decision = await self.decide(obs, task_description)

            # 高風險 → 等確認
            if decision.needs_confirmation:
                await self.request_confirmation(decision)
                continue

            # 信心太低 → escalate
            if decision.confidence < 0.3:
                return await self.escalate("confidence too low", obs)

            # ═══ EXECUTE ═══
            result = await self.execute(decision)

            # 執行失敗 → fallback chain（在 decision_engine 裡處理）
            if result.status != CommandStatus.SUCCESS:
                fallback_result = await self.engine.execute_with_fallback(
                    task=decision, caps=obs.device_state, agents=self.sender
                )
                if not fallback_result.success:
                    return await self.escalate("all modes failed", obs)
                result = fallback_result

            # ═══ VERIFY ═══
            verify = await self.verify(decision, result)

            if verify.passed:
                self.step_history.append({
                    "step": self.step_count,
                    "action": decision.next_action.value,
                    "result": "success",
                    "evidence": verify.evidence
                })
                remaining.pop(0)  # 完成一步，移除
            else:
                # 驗證失敗 → retry 同一步
                self.step_history.append({
                    "step": self.step_count,
                    "action": decision.next_action.value,
                    "result": "verify_failed",
                    "evidence": verify.evidence
                })
                verify_retries = sum(
                    1 for h in self.step_history[-3:]
                    if h["result"] == "verify_failed"
                )
                if verify_retries >= MAX_VERIFY_RETRIES:
                    return await self.escalate("verify failed 2x", obs)

            self.step_count += 1

        # ═══ LOOP EXIT ═══
        if not remaining:
            return await self.complete()
        else:
            return await self.escalate("max steps exceeded", None)

    async def observe(self, remaining: list[str]) -> Observation:
        """截圖 + 裝置狀態 + 歷史組合。"""
        screenshot = await self.sender.send_command(ActionType.BROWSER_SCREENSHOT, {})
        state = await self.sender.get_device_state()
        return Observation(
            screenshot_b64=screenshot.result.screenshot_b64,
            page_url=screenshot.result.page_url,
            page_title=screenshot.result.page_title,
            device_state=state,
            step_history=self.step_history,
            remaining_steps=remaining
        )

    async def decide(self, obs: Observation, task: str) -> Decision:
        """LLM 看 observation → 決定下一步 action。"""
        prompt = self.build_decide_prompt(obs, task)
        llm_resp = await self.planner.decide_next_action(prompt)
        # 安全檢查
        if self.is_dangerous(llm_resp.next_action, llm_resp.params):
            llm_resp.needs_confirmation = True
        return llm_resp

    async def verify(self, decision: Decision, result) -> VerifyResult:
        """執行後截圖 + 比對，確認 action 真的生效。"""
        await asyncio.sleep(SCREENSHOT_DELAY_MS / 1000)
        after = await self.sender.send_command(ActionType.BROWSER_SCREENSHOT, {})
        return await self.verifier.check(
            action=decision.next_action,
            params=decision.params,
            before=result,
            after=after
        )

    async def escalate(self, reason: str, obs) -> dict:
        """升級人工 — 截圖 + 上下文 → Discord。"""
        self.session.status = SessionStatus.ESCALATED
        await self.sender.notify_discord(
            f"⚠️ Remote task escalated: {reason}\n"
            f"Steps completed: {len(self.step_history)}\n"
            f"Device: {self.session.device_id}"
        )
        return {"status": "escalated", "reason": reason, "steps": self.step_history}

    async def complete(self) -> dict:
        """任務完成 — 最終截圖 + 報告。"""
        final = await self.sender.send_command(ActionType.BROWSER_SCREENSHOT, {})
        return {
            "status": "completed",
            "steps_total": self.step_count,
            "step_history": self.step_history,
            "final_screenshot": final.result.screenshot_b64
        }

Loop 與其他模組的關係

# 元件關係圖

Kai API (POST /api/remote/task)
    ↓
TaskQueue (排隊 + 優先級)
    ↓
AgentLoop.run_task() ← 核心循環，本 Section 定義
    │
    ├── observe()   → WebSocket → Client Daemon → 截圖/狀態
    ├── decide()    → DecisionEngine (Section 11) → LLM Planning
    ├── execute()   → WebSocket → Client Daemon → 執行
    │                  → DecisionEngine.execute_with_fallback()
    └── verify()    → WebSocket → 截圖 → ScreenDiff
                        → 驗證策略（按 action type 分）
    ↓
完成 / 升級人工
    ↓
SessionManager (Section 13) → 更新 session 狀態

# 關鍵原則：
# 1. AgentLoop 不直接操作裝置 — 所有操作透過 WebSocket Command
# 2. AgentLoop 不做模式選擇 — 交給 DecisionEngine
# 3. AgentLoop 只管「循環邏輯」= observe + decide + execute + verify
# 4. 一個 AgentLoop instance 對應一個 Session（不跨裝置）
# 5. 多裝置 = SessionManager 管多個 AgentLoop instance
安全限制（硬編碼）：
- 單任務最多 50 步（超過強制 escalate，防 AI 無限循環）
- 同一步驟驗證失敗最多重試 2 次（第 3 次 escalate）
- needs_confirmation 的 action 會暫停整個 loop，直到 Rain 在 Kai UI 點確認
- LLM confidence < 0.3 → 直接 escalate（不嘗試執行）

Queue Worker 邏輯

# 排程器：每個裝置一個 worker，保證同裝置指令順序執行

class CommandQueueWorker:
    def __init__(self, device_id: str):
        self.device_id = device_id
        self.running = False

    async def process_next(self):
        # 1. 從佇列取下一個 pending 指令（同裝置，按 priority + created_at 排序）
        cmd = await db.fetch_one("""
            SELECT * FROM remote_command_queue
            WHERE device_id = ? AND status = 'pending'
            ORDER BY priority ASC, created_at ASC
            LIMIT 1
        """, [self.device_id])
        if not cmd: return

        # 2. 標記 running
        await db.execute("UPDATE ... SET status='running', started_at=?")

        # 3. 透過 WebSocket 發送給 Client Daemon
        response = await self.send_to_device(cmd)

        # 4. 處理結果
        if response.status == "success":
            await self.mark_done(cmd, response)

        elif cmd.retry_count < cmd.max_retries:
            # 重試：指數退避
            backoff = [1, 3, 10][min(cmd.retry_count, 2)]
            await asyncio.sleep(backoff)
            await db.execute("""
                UPDATE remote_command_queue
                SET status='pending', retry_count=retry_count+1, error=?
                WHERE id=?
            """, [response.error, cmd.id])

        else:
            # 超過重試次數 → 進 Dead Letter Queue
            await self.move_to_dlq(cmd, response)
            # 嘗試 fallback 模式
            if cmd.on_failure == "fallback_vision":
                await self.enqueue_vision_fallback(cmd)
            elif cmd.on_failure == "escalate":
                await self.escalate_to_human(cmd)

    async def move_to_dlq(self, cmd, response):
        """移入 Dead Letter Queue + Discord 通知"""
        await db.execute("INSERT INTO remote_command_dlq ...")
        await db.execute("UPDATE remote_command_queue SET status='dlq' WHERE id=?")
        await discord.notify(
            f"⚠️ 指令失敗 3 次進入 DLQ\n"
            f"裝置: {cmd.device_id}\n"
            f"動作: {cmd.action}\n"
            f"錯誤: {response.error}"
        )

SessionManager — 多機管理核心

# backend/app/services/remote_session_manager.py

import asyncio
from dataclasses import dataclass, field
from typing import Dict, Optional

MAX_CONCURRENT_SESSIONS = 5

@dataclass
class RemoteSession:
    session_id: str
    device_id: str
    status: str = "connecting"     # 狀態機（見 Section 09）
    mode: str = "auto"
    ws_connection: Optional[object] = None
    queue_worker: Optional[object] = None
    screenshot_buffer: list = field(default_factory=list)
    created_at: str = ""
    last_activity: str = ""

class SessionManager:
    def __init__(self):
        self.sessions: Dict[str, RemoteSession] = {}
        self.device_locks: Dict[str, asyncio.Lock] = {}

    async def connect(self, device_id, mode, task_desc, operator) -> RemoteSession:
        # 1. 檢查並行上限
        active = [s for s in self.sessions.values()
                  if s.status not in ("closed", "failed")]
        if len(active) >= MAX_CONCURRENT_SESSIONS:
            raise RuntimeError(
                f"已達並行上限 {MAX_CONCURRENT_SESSIONS}。"
                f"活躍 sessions: {[s.device_id for s in active]}"
            )

        # 2. 同一裝置不能開兩個 session
        for s in active:
            if s.device_id == device_id:
                raise RuntimeError(f"裝置 {device_id} 已有活躍 session: {s.session_id}")

        # 3. 建立 session
        session = RemoteSession(
            session_id=self.generate_id(),
            device_id=device_id,
            mode=mode,
        )
        self.sessions[session.session_id] = session

        # 4. 等待 Client Daemon WebSocket 連線
        await self.wait_for_device(session, timeout=30)

        # 5. 啟動該裝置的 Queue Worker
        session.queue_worker = CommandQueueWorker(device_id)
        asyncio.create_task(session.queue_worker.run())

        return session

    async def execute_parallel(self, tasks: list[dict]):
        """多機並行執行：每台裝置的任務獨立排程"""
        # 按 device_id 分組
        by_device = {}
        for task in tasks:
            by_device.setdefault(task["device_id"], []).append(task)

        # 每台裝置的任務序列，跨裝置並行
        async def run_device_tasks(device_id, device_tasks):
            for task in device_tasks:
                await self.enqueue_command(task)
            return await self.wait_all_complete(device_id)

        results = await asyncio.gather(
            *[run_device_tasks(did, dtasks)
              for did, dtasks in by_device.items()]
        )
        return dict(zip(by_device.keys(), results))

重點：
1. 每台裝置 獨立 WebSocket（互不影響）
2. 每台裝置 獨立 Queue Worker（指令不混）
3. 同裝置指令序列執行（避免衝突）
4. 跨裝置指令並行執行（asyncio.gather）
5. 任一裝置失敗 不影響其他裝置
