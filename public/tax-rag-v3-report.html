Debug 紀錄

本次修復的 3 個 Bug

1
fts_vector 全為 NULL
問題
migrate_to_v3.py 遷移了 2,534 筆 document_chunks，但沒有 populate fts_vector 欄位，導致 PostgreSQL FTS 完全失效，Hybrid Search 只剩向量分支在運作。
修復
UPDATE document_chunks SET fts_vector = to_tsvector('simple', content)
影響
Hybrid Search 恢復 PG FTS 分支，檢索精準度提升
2
_rerank 讀錯 score 欄位
問題
Rule Engine 注入的 chunks 只有 rrf_score: 1.0，但 _rerank 讀取 item.get("score", 0.0)，導致強制注入的關鍵法條分數歸零，被排到結果最後。
修復
改為 base_score = item.get("rrf_score") or item.get("score", 0.0)
影響
外國專業人才延攬及僱用法等法條現在正確排在前位
3
殭屍 uvicorn 進程遮蓋新程式碼
問題
LaunchAgent KeepAlive 讓舊進程持續運行，新改的 code 永遠不會被載入。前兩個 bug 的修復全部無法生效，排查花了最多時間。
修復
kill -9 舊 PID + launchctl bootout 停止 LaunchAgent，改為手動 nohup 運行。
影響
此 Bug 是根本阻斷點，解決後前兩個修復立即生效

架構設計

系統流程

1
FactExtractor
提取事實、偵測缺失資訊，為後續檢索準備結構化查詢。
2
v3 HybridSearch
三路並行，RRF（k=60）融合後取 top-20。
Qdrant 向量搜尋（text-embedding-3-small）
PG FTS ts_rank（plainto_tsquery simple）
RRF 融合
3
Rule Engine
強制注入關鍵法條（外國專業人才 / OBU / 扣繳 等），確保核心法源不會遺漏。
4
Rerank
keyword_score × 0.30 + rrf_score × 0.67 + doc_type × 0.03，取 top-10。
5
Reasoner Router
依問題類型分流至對應 Reasoner。
legal — 法規解釋
filing — 申報流程
accounting — 會計分錄
exam — 考題練習
6
LLM Reasoning
Claude claude-3-7-sonnet-20250219，輸出 8 欄結構化格式。
7
Legal Basis Validator + 信心分數
驗證法律根據，計算信心等級（low / medium / high），附 sources / confidence / disclaimer 完整回應。
