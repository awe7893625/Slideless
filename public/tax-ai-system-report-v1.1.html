Router 類型 觸發詞 例題
law_lookup 規定、期限、條文、怎麼處理 「非營利事項免稅的相關規定」

效果：路由精度提升 → 下游 AI 選型更準確（法律推理用 Opus，計算用精確 API）

改動 02

法源層級改為彈性規則

從「強制全部湊齊」到「風險級聯」
舊設計（v1.0）

強制需求：每回答必須有

  • ≥1 母法
  • ≥1 解釋令/函釋
  • ≥1 QA 出處
新設計（v1.1）

彈性規則：

  • 必須有：≥1 直接法源（母法 or 權威裁定）
  • 條件補充：高風險題（eligibility、稅額、複雜身分）→ +≥1 輔助法源
  • 排除條件：不強制湊 QA，若品質不夠就不補
if confidence == 'high' and direct_source_found: return answer_with_single_source() elif confidence == 'medium' and risk_level == 'high': return answer_with_direct_source + supplementary_sources() elif confidence == 'low': return answer_with_warning + missing_facts()

效果：避免「為了湊法源而牽強引用 QA」，保持答案品質優先。

改動 04

Benchmark 改三層驗證

從單一選擇題到完整評估體系

Layer 1：近兩年選擇題（標準化評估）

  • 記帳士 + 會計師考試，25-30 題/年
  • 評分：accuracy、citation_accuracy、year_correctness、missing_direct_source

Layer 2：申論題 / 實例題（複雜推理）

  • 會計師考申論題，10-15 題/2 年
  • 評分：fact_extraction、judgment_correctness、accounting_vs_tax_clarity

Layer 3：高頻實務題（貼近真實）

  • 自建 10-20 題（外籍專才、房租補助、交際費、非營利事項）
  • 評分：文件完整度、風險提示、用戶可執行性

改動 05

加「Fact Extraction 層」

新增事實抽取，自動識別缺失資訊
Input
使用者提問
「外籍專才月薪 100 萬，公司想給子女獎學金算福利費」
Processing
Fact Extractor 識別
entities: "外籍專才", "子女", "獎學金" tax_year: 2024 subject_type: "個人" + "公司" nationality: "外籍" known_facts: ["月薪100萬", "子女", "獎學金"] missing_facts: ["子女年紀", "獎學金性質", "核發公司是誰", "專才簽證類別"] risk_flags: ["外籍身分特殊稅務", "親子關係福利", "免稅額上限"]
Output
信心度級聯
缺失事實 0 個：confidence = high
缺失事實 ≤2 個：confidence = medium → 加警告「以下假設為...」
缺失事實 ≥3 個：confidence = low → 拒絕或標記嚴重缺失
新表：fact_extraction_log
  • query_id, entities, context (year/type/nationality)
  • known_facts[], missing_facts[], risk_flags[]
  • confidence_impact: 缺失事實如何影響信心度

效果：透明化「為什麼信心度低」，用戶清楚知道需要補充什麼。
