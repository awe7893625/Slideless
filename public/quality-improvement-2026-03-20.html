CURRENT STATE

現狀診斷

近 20 筆任務評分平均 4.4/10，低於閾值 6.0。根據 lessons.md 和 session review 數據，發現根本問題五個。

ROOT CAUSES

根本原因分析

品質下降的五個核心問題，影響範圍涵蓋流程、測試、追蹤和溝通層面。

1. Session Review 機制失效

require-session-review.sh Bash heredoc bug → 751 次 hook 全自動 approve → 合規率 20%

2. 派發分工規則未執行

Major 任務（3+ 檔案）應 dispatch，但大量由單人完成 → Quill 缺席、Silo 缺席

3. 測試框架空殼

CI 只驗「有跑」不驗「質量」→ 假測試照樣 PASS → 覆蓋率 ≈ 0

4. 工作指令未登記

Session 沒立即建 Kai task → 無追蹤、無統計、無透明度

5. Prompt 指導不足

Scope Card（R-00）存在但未強制執行 → Major 等級判斷混亂

LAYER 1: PROCESS

改善方案：流程硬規則

立即可執行的三項流程修正，預期 3-5 天內見效。

第一個工作指令強制登記

  • Session 開始 → 立即 curl POST /api/tasks（auto_start: true）
  • 無 task = session 無法 exit（hook 強制擋住）
  • 預期效果：+15 分

Major 任務強制分工

  • 3+ 檔案 = Major → 必須 dispatch
  • Code → Quill、SEO → Silo、Arch → Ray
  • 預期效果：+20 分

測試品質把關（標準已定義）

  • 每 API endpoint = happy path + ≥1 error case
  • 空測試/假測試 = Quill Blocker
  • 預期效果：+15 分
LAYER 2: PROMPTS

改善方案：Prompt強化

強化協作提示，鞏固流程硬規則，1-2 週內見效。

Scope Card 前置檢查

  • 自動提示：有新任務嗎？
  • 錯誤 → 提醒改正

Session Review 強制完成

  • 停在 /compact 前必須完成五問+反思
  • 不能空答（hook 檢查）

SKILL_DELTA 自動升級

  • Session 發現 → 自動抓入 speed-improvements.md
  • Quill 72h 內升級
LAYER 3: AUTOMATION

改善方案：自動化防護

長期制度化監控，4 週內完成部署，持續防止品質滑落。

Task Board 完成率追蹤

  • 夜班 Module J 統計 dispatch rate
  • < 50% → Discord 通知 Klay

Health Check 品質信號

  • 日檢：review 合規率、test pass rate、dispatch rate
  • 任一 < 60% → 建 quality-review task

自動回退機制

  • 評分連 3 天下降 → 暫停新任務
  • 強制做「品質改善周」
