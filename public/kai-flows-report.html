05
技術修正（Quill 審查通過）
CRITICAL — 2/2 已修復
RESOLVED _action_code 沙盒安全：__builtins__ 白名單限制
任何 workflow 使用者都可透過 import os 執行任意 shell 命令，權限逃逸風險極高。
修正後：__import__ 已阻擋；白名單 20 個安全函數 abs / len / str / int / float / bool / list / dict / tuple / set sum / min / max / sorted / range / enumerate / zip / map / filter / round
RESOLVED workflow/create 端點 API Key 認證保護確認
確認 workflow 建立端點已正確啟用 API Key 驗證中介層，未授權請求無法建立惡意 workflow。
HIGH — 3/3 已修復
RESOLVED proc.kill() 後加 await proc.wait() — 清除殭屍程序
子程序 timeout 後直接 kill 但不等待回收，長期累積導致殭屍程序（zombie process）佔用系統資源。
kill() → kill() + await proc.wait() — 確保子程序完全回收
RESOLVED asyncio.get_event_loop() → asyncio.get_running_loop()
Python 3.10+ 棄用 get_event_loop()，在協程中呼叫會產生 DeprecationWarning，未來版本將直接報錯。
全面替換為 asyncio.get_running_loop()，符合 Python 3.10+ 規範
RESOLVED fetchHistory 加 res.ok 檢查 — 前端 HTTP 錯誤處理
前端 fetchHistory 函數未檢查 HTTP response 狀態碼，伺服器 4xx/5xx 錯誤時仍嘗試解析 JSON，導致靜默失敗難以除錯。
加入 if (!res.ok) throw new Error(`HTTP ${res.status}`) 前置檢查
MEDIUM — 已修復
RESOLVED _action_sheets：sa_json 禁止從 step 傳入
防止 Google Service Account 私鑰被存入 workflow JSON 定義或執行日誌，避免私鑰洩露至資料庫或日誌系統。
sa_json 僅允許從 GOOGLE_SA_JSON 環境變數讀取 step 層傳入的 sa_json 欄位將被忽略並記錄警告

P0 Bug Fix：_action_command & r2_upload
_action_command — asyncio 化
之前（阻塞 event loop）
subprocess.run(cmd, shell=True) # 同步阻塞，整個 asyncio # event loop 卡死期間無法 # 處理其他請求
之後（真非同步）
proc = await asyncio .create_subprocess_shell(cmd) await asyncio.wait_for( proc.wait(), timeout=t) # timeout 後： proc.kill(); await proc.wait()
_action_r2_upload — SigV4 修正
之前（永遠 403）
httpx.put(url, auth=BasicAuth(id, secret)) # R2 不支援 HTTP Basic Auth # → 永遠回傳 403 Forbidden
之後（boto3 SigV4）
s3 = boto3.client('s3', endpoint_url=endpoint, region_name="auto", config=Config( signature_version="s3v4")) loop.run_in_executor( None, _upload) # 非同步包裝
