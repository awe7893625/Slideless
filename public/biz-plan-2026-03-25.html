🚀 AgentDesk

AI Agent 團隊管理平台（聊天優先 + 本地優先）

目標定位：
「你已經有 Claude Code + Codex CLI 訂閱，AgentDesk 幫你把它們快速配置成一支 AI 團隊」

定位： 目前缺乏「聊天優先」的 AI Agent 團隊管理工具。Conductor 免費但功能不全，需要正面但有差異的替代品。

預估月收
US$3,500-8,500
M1（早期）/ US$15k+ M3
MVP 時程
3-4 週
已有 80% 基礎技術

問題定義

現狀痛點

  • 工具分散： 用戶同時訂閱 Claude Code + Codex CLI，但無統一界面，手動切換浪費時間
  • 缺聊天入口： Conductor 是 IDE，無 Telegram/Discord 聊天界面（mobile-first 使用者無法用）
  • 無本地方案： 現有工具都是雲端，使用者不願上傳 repo 到外部
  • 團隊配置複雜： 手動寫 config，沒有「一鍵配置」的安裝體驗

市場機會

  • TAM： Claude Code + Codex 訂閱用戶 = 估計全球 50k-100k（Anthropic 企業用戶 + 開發者）
  • SAM： 想同時管理多個 AI Agent 的工程團隊 = 估計 5k-10k
  • SOM： Year 1 目標 50-100 付費用戶 = 每月 US$3.5k-8.5k
差異化： Conductor（IDE）vs AgentDesk（聊天 + 本地），兩個不同的用戶需求，非正面競爭

解決方案

核心架構（用戶自訂 API Key）

三入口
• Web Dashboard（GUI）
• Telegram Bot
• Discord Bot
一後端
• 本地伺服器 + Docker
• FastAPI
• PostgreSQL + Redis

用戶情境（四種都支援）

情境 用戶有什麼 AgentDesk 配置
A Claude Code 訂閱 Claude Code 團隊
B Codex CLI 訂閱 Codex CLI 團隊
C 兩者都有 混合團隊（最強）
D API Key（Anthropic / OpenAI） 直接 API 模式

安裝體驗（目標：單一指令）

curl -fsSL https://get.agentdesk.dev | bash

自動偵測環境 → 互動填 API Key → 配置團隊 → 起服務 → 開瀏覽器
