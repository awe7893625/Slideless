Root Cause

三條鏈互搶同一顆 token

Conflict 01

OpenClaw tax account

`openclaw.json` 內 `channels.telegram.accounts.tax.enabled = true`，gateway log 可見 `[telegram] [tax] starting provider (@ClaudeLinXI_bot)`。

Conflict 02

M4 tax_bot.py

`tax_bot.py` 本身也在 long polling，同時寫入 `~/.local/log/tax-bot.log`，連續出現 `409 Conflict`。

Conflict 03

claw-claude-proxy john bot

`.env.john` 也綁了同一顆 `ClaudeLinXI_bot` token，`com.rain.john-telegram.plist` 仍可能被 launchd 拉起。

Fixes

這次實際做的修補

01

停用 OpenClaw tax Telegram account

在 `openclaw.json` 把 `channels.telegram.accounts.tax.enabled` 改為 `false`，讓 OpenClaw 不再接手這顆 bot token。

02

強化 tax_bot.py 啟動與單實例

加入 file lock、proxy、快速 socket health check、`/ping`、初始化階段 log，並改善重連流程，避免之後重啟又自己打架。

03

移除舊 john tax bot LaunchAgent

把 `com.rain.john-telegram.plist` 移到 `_disabled/`，只保留新的 `com.rain.tax-bot` 路徑。

openclaw.json channels.telegram.accounts.tax.enabled = false tax_bot.py - single instance lock - OPENCLAW_PROXY_URL = http://127.0.0.1:18790 - socket health check - /ping command - startup logs: initialize / deleteWebhook / polling
