API 設計 POST /api/tax/query

Request 
// POST /api/tax/query
{
  "question": "子女獎學金算員工薪資嗎？",
  "tax_type": ["所得稅"],
  "session_id": "telegram_chat_id"
}
Response（六段式） 
{
  "success": true,
  "answer": {
    "summary": "原則上不算薪資，屬員工福利費",
    "legal_basis": [{
      "law_name": "外籍專業人士租稅優惠之適用範圍",
      "ruling_no": "台財稅字第11204616990號令",
      "source_url": "https://law-out.mof.gov.tw/...",
      "doc_type": "regulation"
    }],
    "explanation": "詳細說明...",
    "practical_steps": ["實務步驟..."],
    "caveats": ["注意事項..."],
    "disclaimer": null
  },
  "confidence": "high"
}
🔒 Legal Basis Validator（Ray 條件 A）
  • ✦ 格式驗證：legal_basis 必須符合「法律名稱第X條」或「令字號」pattern
  • ✦ 空字串 / 格式不合 → 強制 fallback：「查無明確法源，建議諮詢專業稅務師」
  • ✦ 沒有法源就不能輸出確定結論（核心安全規則，不可繞過）

資料模型 Database Schema

📋 tax_laws（法規文件主表）
-- 主鍵與識別
id              UUID PK
doc_id          VARCHAR UNIQUE
source          VARCHAR  -- mof / moj / ntbt
source_url      VARCHAR NOT NULL

-- 法規資訊
law_name        VARCHAR
article_no      VARCHAR  -- 第 X 條
ruling_no       VARCHAR  -- 令字號
publish_date    DATE
effective_date  DATE
status          ENUM(active, superseded, repealed)

-- 內容
content         TEXT
raw_file_path   VARCHAR  -- ~/.m4_data/tax_docs/

-- 分類與向量
tax_type        TEXT[]
topic_tags      TEXT[]
keywords        TEXT[]
embedding_status ENUM(pending, done, failed)
🔍 tax_chunks（RAG 基本單位）
  • ✦ 以「條」為原子單位，不跨條合併
  • ✦ 超過 2000 token 的長條，以「款」為子單位切割
  • metadata：law_name, article_no, paragraph_no, doc_type, publish_date
📝 tax_queries（查詢 log）
  • ✦ 完整記錄：提問、top-10 chunks、推理輸入
  • ✦ 記錄：完整答案、信心分數、使用模型
  • ✦ 用途：品質追蹤 + 引用溯源 + 效能分析

Klay 裁決 4 個關鍵決策

問題 1 — 向量庫選型
Phase 1 用 LightVectorStore
Ray 建議先驗證 RAG 品質，Phase 2 再升 Supabase pgvector。避免初期過度工程化，讓系統更快落地驗證。
問題 2 — Phase 1 稅種
3 部核心法律
所得稅法 + 加值型及非加值型營業稅法 + 外籍專業人才延攬及僱用法。涵蓋最高頻問題，可在 2 週內完成入庫。
問題 3 — PDF 儲存策略
本地路徑儲存
路徑存 DB，檔案放 ~/.m4_data/tax_docs/。避免 SQLite BLOB 膨脹，保持資料庫輕量。
問題 4 — Telegram 使用者
Rain 專用（初期）
法律諮詢系統需先驗證品質再開放。Phase 1 完成品質驗證後，再評估開放給更多使用者。

Opus 雙模型覆核規則 Ray 條件 B

觸發條件（任一符合即啟動）
1
信心分數 < 0.60：RAG Reasoning 階段低置信度
2
高風險關鍵字：「稅務規避」「移轉定價」「OBU」「跨國」「境外所得」「特殊關係」
3
估算金額 > NT$500 萬：問題中含金額且超過閾值
限制與分歧處理
每日上限
20 次 Opus 覆核。超過後降級為 Sonnet + 警告使用者「建議諮詢專業稅務師」
覆核分歧處理
Opus 與 Gemini 結論不一致 → confidence 強制設為 low，輸出兩種觀點供使用者判斷

爬取來源 官方來源 · 驗證合規

來源 URL robots.txt 狀態 資料類型
全國法規資料庫 https://law.moj.gov.tw ✅ 允許主路徑爬取 law / regulation
財政部 https://www.mof.gov.tw ✅ 允許主路徑爬取 notice / ruling
台北國稅局 https://www.ntbt.gov.tw ✅ 允許主路徑爬取 qa / notice
財政部賦稅署 https://www.dot.gov.tw ⏳ 待確認 regulation / ruling
行政院公報 https://gazette.nat.gov.tw ⏳ 待確認 law
⚖️ 法律合規原則：僅爬取公開政府法規資料；遵守各站 robots.txt；不建立超量請求（rate limit 設定 1 req/3s）；資料僅用於法規問答研究目的。
