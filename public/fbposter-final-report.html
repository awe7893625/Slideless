Section 01

產品簡介

社團發文助手是專為台灣房仲設計的 Chrome 擴充功能，解決房仲每天需要手動在數十個 FB 社團重複貼文的痛點。

核心價值

台灣房仲每天至少花 1-2 小時在各社團貼房源資訊。社團發文助手透過 591/HouseBox 自動擷取、Spintax 智慧文案變化、mbasic 快速發文引擎，讓原本 2 小時的工作縮短到 10 分鐘以內。

步驟一：匯入物件

三種匯入方式：貼上 591/HouseBox 連結批量擷取（最多 30 筆）、PDF 批量匯入（整個資料夾）、或手動建立。系統自動抓取標題、價格、坪數、格局、圖片等完整資訊。

步驟二：智慧文案生成

擷取後自動套用專業房仲模板產生發文內容，包含物件規格、賣點自動分析（近捷運、邊間採光佳等）、業務聯繫資訊與地區 Hashtag。支援 Spintax 語法讓每篇文都不同。

步驟三：選擇社團 & 一鍵發文

從已加入的社團清單勾選目標社團（支援「社群包」一鍵套用整組），點擊「開始發文」後系統自動逐一開啟 mbasic.facebook.com 頁面、填入文案、提交發佈，每篇 3-5 秒。

步驟四：追蹤 & 自動刪文

發文後自動記錄每則貼文的社團、時間、狀態、連結。可設定 N 天後自動刪除（預設 3 天），由 Chrome Alarm API 定時觸發，避免社團管理員認定洗版。

Section 03

11 個功能頁面導覽

完整後台截圖與功能說明。每個功能模組都是為房仲日常發文流程量身打造。

01
首頁 Dashboard

進入擴充功能的第一個畫面。上方四大 KPI 數字即時顯示：今日發文數、成功率、已管理社團數、佇列中待發數量。中間是近 7 天發文趨勢圖表，下方提供「一鍵發文」「建立排程」「管理社團」三個快速操作按鈕，讓使用者不需翻找就能直接開始工作。

02
帳號管理

管理 Facebook 帳號 Profile 與登入資訊。支援多帳號切換（帳號 Profile），每個 Profile 獨立管理社團清單和名片資訊。FB 帳號密碼僅存於本機瀏覽器（chrome.storage.local），永不外傳。下方「我的名片」可填入姓名、電話、LINE、公司名稱等，591 匯入時會自動替換為你的聯絡方式。

03
文案設定

管理發文模板，支援 Spintax 隨機文案語法。例如輸入 {出售|出租}，每次發文時系統會隨機選擇其中一個詞。可建立多組模板，搭配最多 3 組不同文案輪替，產生無限種組合，避免 FB 偵測為重複內容。模板名稱方便辨識（如「中和出售」「信義租屋」），一鍵套用到發文佇列。

04
一鍵發文

核心功能頁面。上方支援「批量匯入 591/HouseBox」，貼上連結後自動擷取房源資料（最多 30 筆），或選擇 PDF 檔案/整個資料夾批量匯入。下方可手動建立發文，填入物件標題、內容、圖片。所有匯入的物件會進入發文佇列，勾選目標社團後一鍵開始自動發文。

05
社團管理

搜尋、加入並管理 FB 社團清單。提供關鍵字搜尋社團功能，搜尋結果可一鍵申請加入並自動存入清單。內建推薦房仲社團庫，一鍵加入台灣最大的房屋買賣社團。支援「社群包」分組管理，把社團分成「台北租屋」「新北買賣」等群組，發文時一鍵套用整組社團。瀏覽 FB 社團頁面時，Content Script 會自動偵測並提供加入按鈕。

06
排程發文

設定自動發文時間，支援每天定時發文。在「一鍵發文」頁面的佇列項目上點擊排程按鈕，選擇日期和時間後系統會在指定時刻自動開始發文。排程任務清單顯示在此頁面，可檢視待執行、已完成的排程。底層使用 Chrome Alarm API 實現精確排程觸發。

07
Marketplace 上架

一鍵上架物件到 Facebook Marketplace。佇列中的物件可同時發到社團和 Marketplace，擴大曝光管道。系統會自動開啟 Marketplace 建立商品頁面，填入標題、價格、描述、圖片等資訊，對有在 Marketplace 發布物件的房仲特別實用。

08
自動排程

設定全自動化任務的中控台。四大自動任務一覽：自動社團發文、自動加入社團、自動刪文、熱門貼文留言，每項顯示「運行中」或「未設定」狀態。下方可調整每篇發文間隔（預設 3 分鐘，建議 2-3 分鐘避免被 FB 限制）、自動刪文天數、文案微變化開關、智慧延遲（根據社團數量自動調整間隔）與發文完成通知。

09
自動養號

模擬真人使用 Facebook 的行為，降低帳號被偵測為自動化的風險。包含兩大功能：「自動留言」對社團最新貼文自動送出留言（支援 Spintax 語法讓每則留言不同），可設定每社團留言數量；「自動養號」模擬真人滑動、瀏覽、按讚等操作，有一般模式（10 分鐘）可選。提升帳號互動率，讓 FB 演算法認定為正常用戶。

10
數據統計

查看所有發文記錄，管理刪除排程。頂部統計面板顯示總發文篇數、已發/待刪/失敗/已刪各狀態數量。支援按狀態篩選（全部、已發、待刪、失敗、已刪）和搜尋社團名稱或文案內容。表格欄位包含：社團名稱、文案摘要、發文時間、狀態、操作。提供「全部立即刪除」快速清理功能，以及清除失敗/已刪記錄的快捷操作。

11
系統設定

授權碼驗證、防偵測設定與資料管理。授權碼輸入區支援 FBP-XXXX-XXXX-XXXX 格式驗證，分為基本版/專業版/旗艦版方案。「發文操作畫面」開關可選擇是否顯示自動開分頁的過程。「進階防偵測」啟用指紋遮罩（包含 WebDriver 隱藏、Canvas 指紋微擾、WebGL 資訊模糊、語言一致性），建議保持開啟。資料管理區提供匯出/匯入/清除所有資料功能。

Bonus

實際發文畫面

系統自動在 FB 社團建立貼文的真實截圖，包含從 HouseBox 擷取的完整物件資訊。

A
FB 社團自動建立貼文

系統在 www.facebook.com 開啟社團的「建立貼文」對話框，自動填入物件標題（含 emoji 吸引眼球）、地址（安全處理只到路名）、價格、物件規格、自動分析的賣點、聯繫資訊、Hashtag，並上傳物件圖片。右側可看到社團內其他房仲的手動貼文作為對比。

B
HouseBox 物件原始頁面

好屋網（HouseBox）的物件資訊頁面，系統會自動擷取：標題、地址、價格、單價、坪數、格局、樓層、建物類型、業務聯繫電話等完整資料，並轉換為專業的 FB 發文格式。

Section 05

技術亮點

底層工程細節，理解為什麼社團發文助手比其他工具更快、更穩、更安全。

chrome.debugger API

當 Facebook 的 React 層攔截普通 JavaScript 點擊事件時，系統自動切換到 chrome.debugger API，在系統層級模擬滑鼠事件（mousePressed / mouseReleased / mouseClicked），繞過 React 事件系統的防禦。這是處理 www.facebook.com 圖片上傳按鈕的關鍵技術。

waitForCondition 速度優化

自研的條件等待引擎，以 300ms 為間隔、5 秒超時上限，透過 chrome.scripting.executeScript 注入條件函數到目標頁面。取代傳統的固定等待（sleep 10 秒），讓 DOM 就緒就立刻繼續，實現 mbasic 發文 3-5 秒/篇的極速表現。

6 層指紋遮罩防偵測

每次發文前注入的 stealth patches：(1) navigator.webdriver 隱藏 (2) Plugins 列表偽造 (3) Canvas 指紋微擾（改變幾個像素改變雜湊但不影響視覺）(4) WebGL 渲染器資訊模糊 (5) 語言偏好一致性 (6) Chrome Runtime 物件確保。讓 FB 的自動化偵測系統無法辨識。

mbasic 雙軌發文引擎

無圖片時走 mbasic.facebook.com（純 HTML，無 React/JS 框架，DOM 穩定，3-5 秒完成）；有圖片時走 www.facebook.com 的 compose dialog（支援圖片上傳）。www 失敗時自動 fallback 到 mbasic 純文字版，確保發文成功率。

Spintax 巢狀展開引擎

支援 {{A|B}|C} 巢狀語法，從最內層開始展開。搭配 3 組文案輪替，產生的組合數趨近無限。每篇發到不同社團的文案都是獨一無二的，FB 的重複內容偵測演算法無法命中。有完整的 Jest 單元測試覆蓋。

社團規定自動偵測

新增社團時，系統在背景開啟該社團的 /about 頁面，用 Content Script 抓取發文規定文字（如「必須用【出售文】開頭」「禁止發連結」等）。發文時自動依照每個社團的規定調整格式，降低被管理員刪文的風險。

技術規格總覽

  • Chrome Extension Manifest V3 架構，支援最新 Chrome 瀏覽器
  • 純前端實作，零後端依賴，所有資料存於 chrome.storage.local
  • 總程式碼量約 9,800 行（background.js 2,836 + popup.js 5,007 + CSS 2,030）
  • 使用 chrome.alarms API 實現排程發文和自動刪文
  • chrome.scripting API 動態注入爬蟲和發文腳本
  • 591 爬蟲需等待 Vue mount（7-10 秒），HouseBox 較快（3-4 秒）
  • PDF 解析使用 pdf.js（pdf.min.mjs + pdf.worker.min.mjs）
  • 授權碼驗證 + Device ID 綁定，防止無限複製使用
  • 3 次自動重試機制（每次間隔 5-8 秒隨機延遲）
