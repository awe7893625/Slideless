發現問題

已修復

#1 中控台 GA4 Property ID 錯位

webspeed 使用了選物事務所的 GA4 ID（526022066），導致 Traffic Dashboard 中留白工作室顯示的是選物事務所的流量數據。

已修復

#2 選物事務所 GA4 未接入中控台

SITE_DISPLAY 中 somethingselect 的 ga4_id 為 None，中控台完全看不到選物事務所的 GA4 流量。

已修復

#3 留白工作室未加入 GSC 追蹤

GSC_SITES 字典中沒有 blankspaces.uk，Search Console 數據未同步到中控台。

已修復

#4 資料庫健康檢查永遠失敗

/api/health/detailed 永遠顯示 database=false、status=degraded。根因：SQLAlchemy 2.x 原始 SQL 需要 text() 包裝。

已修復

#5 序位 TheOrder 缺少 /api/health 端點

API 目錄中沒有 health route，健康檢查腳本無法監控後端。已建立包含 DB + ECPay 設定檢查的 route.ts。

已修復

#6 健康檢查腳本缺少金流端點監控

只檢查首頁 + SEO 文件，不檢查 ECPay webhook、購物車、結帳端點。已新增電商健康檢查區塊。

已修復

#7 排程只跑一次/天，沒有 Email 報告

LaunchAgent 只在 12:50 執行一次，且只寫 JSON 不通知。已升級為 12:00 + 21:00 兩次，加入 HTML Email 自動寄送。

待處理

#8 31 筆安全警報累積未清除

重複的「Kai 安全掃描：風險等級 medium」警報從未被 resolve，不斷累積。需要批次清除 API 或手動處理。

待處理

#9 澄境清潔 robots.txt AI Bot 格式不標準

多個 User-Agent 堆疊在一起，應該每個 bot 獨立一個區塊。

待處理
