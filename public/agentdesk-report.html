產品定位

一句話定位

Cursor 是你的 AI 助手，AgentDesk 是你的 AI 團隊。

目標客群：不會寫程式、只會 vibe coding 的開發者。
痛點：只能跟 1 個 AI 聊天，想同時派多個 agent 做事。

做什麼

同時派多個 AI Agent 並行工作
Web Dashboard 看即時進度
Telegram / Discord 手機下指令
本地跑，用自己的 API Key

不做什麼

不跟 conductor.build 正面打（他是 IDE 工具）
不做 git/merge/PR 教學
不做雲端代管使用者 repo
不需要懂程式就能用

怎麼運作

完整使用流程

1
Telegram 下指令：打「幫我做一個登入頁面」
2
AgentDesk Bot 收到：解析指令 → 建立任務 → 呼叫後端 POST /api/conductor/task
3
Agent 開始工作：Claude Code Agent 在本地 git worktree 執行，每 10 秒回報 heartbeat
4
Dashboard 即時顯示：Agent Live 看板顯示誰在做什麼、做到哪個檔案
5
做完通知你：Bot 推播「John 完成了登入頁面，共修改 5 個檔案」
6
再派第二個 Agent：同時叫另一個 Agent 做後端 API，兩個並行跑

Conductor 四大功能（MS1–MS4）

MS1 — Agent Live Dashboard

每 10 秒 heartbeat 輪詢
顯示：status / current_action / current_files / worktree
60 秒無回報 → 自動標 offline

MS2 — Checkpoint Timeline

每個 Agent turn 建立 git private ref
路徑：refs/conductor/{task}/{turn}
一鍵回滾到任意 checkpoint

MS3 — Diff Viewer

即時 diff vs base branch
純 CSS unified diff，無 Monaco 依賴
一鍵建 GitHub PR

MS4 — Merge Gate

Todo checklist（test/review/docs/manual）
Blocking todo 未完成 → 鎖住 merge
Port allocator：3 ports/workspace
