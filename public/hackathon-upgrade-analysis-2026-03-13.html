2. Skeptic Check

❌ Observer daemon — Ray 否決

「真正的瓶頸不是觀察不夠多，而是搬運不發生。」
27 條 lessons + 8 條 speed-improvements 的資料量足夠，問題在於沒人執行從 speed-improvements.md → SKILL.md 的升級。用一個 hook 修改 + 一個 LaunchAgent 即可修復，不需要 daemon。
  • 安全風險：observations.jsonl 含敏感資料，成為新攻擊面
  • 成本問題：28 agents 觀察量遠超單人使用設計
  • 解錯問題：核心問題是「搬運不發生」，不是「觀察不夠」

✅ Hook Profile — 已實作

Ray 查完 hook-utils.sh 後發現：已有 ECC_HOOK_PROFILE + should_run() + 三級分類表。所有 standard 級 hook 已在使用。只需補一個 is_disabled() 函式（~15 行）。

⚠️ Dynamic Contexts — 有條件通過

不建扁平 contexts/ 目錄。改用 session-start.sh 根據 agent name 注入角色專屬 SKILL.md 摘要 — 更適合 28-agent 組織。開發量 < 半天。
