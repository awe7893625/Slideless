GET /api/taskops/intake-stats 各來源分布統計 + intake_source 欄位填充率分析
🏷
intake_source 欄位 POST /api/tasks 接受新欄位，追蹤任務入口管道
🎛
4 Card Stat Bar IntakeRoutingDashboard.tsx 頂部統計卡片列
🚨
填充率警示橫幅 intake_source 填充率低時自動顯示警告橫幅
IntakeRoutingDashboard.tsx 元件結構
元件區塊 功能說明 資料來源
StatBar 4 個統計卡（總任務數、已標記來源、來源種類數、填充率%） intake-stats API
SourceCards 每個 intake_source 的任務數量 + 佔比條 intake-stats API

UntaggedList 未標記來源的任務清單（可點擊補標） intake-stats API

Phase 4 — 效能分析 + Noel Module N

建立 Agent 效能評分系統，以及夜班 Skill Registry 自動驗證器，讓系統自我健檢。

Agent 效能評分公式
score = base(60) + dispatch_bonus(20) + major_bonus(20)
混合成功定義： taskops_status = DONE（若有追蹤）OR status = success
📊
GET /api/taskops/analytics ?window_days=N，回傳每個 Agent 效能評分
🏆
TaskOpsAnalytics.tsx 7天/30天/90天 分段控制 + 可排序排行榜 + 效能徽章
🌙
noel-module-n.py 夜班 Skill Registry 驗證器，phantom/version/sprawl 三項檢查
🔬
Pure stdlib noel-module-n.py 不引入任何外部依賴，輕量執行
Module N — Skill Registry 三項驗證
驗證項目 說明 觸發條件
phantom Skill 已登錄但對應檔案不存在 找到幽靈 Skill → 記錄警告
version Skill 版本號與 registry 不符 版本漂移 → 提示更新
sprawl 未登錄的 Skill 檔案（孤兒 skill） 超出 registry 管理範圍 → 警告
