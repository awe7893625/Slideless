Section 01

Tax AI 如何擔任顧問？

顧問角色定義 · 完整查詢流程 · 信心度機制 · 知識庫現狀

1-1  顧問角色定義

Tax AI 是一個以台灣稅法知識庫為基礎的 AI 諮詢系統，涵蓋以下四大能力範圍：

  • 稅法條文查詢 — 所得稅法、營業稅法、遺贈稅法等母法及施行細則
  • 實務稅務諮詢 — 外籍人士申報、OBU 境外所得、移轉定價
  • 銀行業會計與稅務 — 金融機構特殊會計處理與稅務規劃
  • 考試題型練習 — 記帳士、會計師、稅務相關證照題型
ℹ
設計原則 所有回答必須附上具體法源引用（母法條文號 + 解釋令字號），不允許無出處的定性陳述。回答信心度依缺失事實數量自動分層，低信心回答強制顯示免責聲明。
1-2  完整查詢流程（6 Steps）
01
3-class Router（規則優先，LLM fallback）
依問題類型分流，規則比對優先，節省 LLM 成本。
tax_type_routing → 加稅種 filter（所得稅 / 營業稅 / 遺贈稅 …）
calculation → 轉 /api/tax-calc（精確計算，不走 RAG）
general → 直接進 RAG Pipeline
↓
02
2-stage Fact Extraction
從問題中抽取關鍵事實，決定信心度上限。
Stage 1: Regex 抽取（年度 / 金額 / 稅種 / 身分）
Stage 2: LLM fallback — 抽到 < 3 個事實時才呼叫
missing_facts_count → 直接影響信心度上限
↓
03
Hybrid RAG Pipeline
向量搜尋 + BM25 關鍵字雙管道，Reranker 法源層級加權。
向量搜尋 — ChromaDB，status=active filter
BM25 fallback — SQLite 全文搜尋
Reranker — 母法 > 施行細則 > 函釋（層級加權）
輸出 — Top-5 chunks 傳入 LLM
↓
04
LLM Reasoning Chain（三層容錯）
三道備援確保服務可用性，全失敗才回傳錯誤。
第一優先： claw-proxy Claude（Claude Code 訂閱）
第二備援： OpenRouter Claude Sonnet
第三備援： Grok-3-mini
全失敗： system_unavailable 錯誤回傳
↓
05
Legal Basis Validator
驗證回答必須包含標準法源格式，否則拒絕輸出。
通過條件： 含「第 X 條」或「台財稅字 XXXXXXX 號」格式
未通過： 回傳 no_legal_basis 錯誤
↓
06
8-column 結構化輸出
標準化回答格式，依模式輸出對應欄位。
summary — 主要結論
legal_basis — 法源清單（母法 / 解釋令 / 函釋層級標示）
explanation — 判斷邏輯
practical_steps — 實務步驟
accounting_treatment — 會計處理（諮詢模式）
answer_key — 標準答案關鍵詞（考試模式）
disclaimer — 低信心警示
disclaimer_footer — 固定免責聲明
1-3  信心度評估機制
等級 條件 意義 建議動作
高信心 missing_facts = 0
法源覆蓋完整		 AI 掌握完整事實，法源明確 可直接參考，建議與稅務師做最終確認
中信心 missing_facts 1–2 個 部分事實缺失，可能影響判斷 補充缺失事實後重新查詢，或直接諮詢國稅局
低信心 missing_facts ≥ 3
或法源不完整		 事實嚴重缺失，無法可靠推論 僅供初步方向參考，須向國稅局正式確認

Section 02

Tax AI 有對外部署嗎？

目前部署狀況 · 使用方式 · 對外開放規劃

2-1  目前部署狀況
⚠
Tax AI 目前僅在本機運行，尚未對外公開 系統已完整可用，但未經 Benchmark 達標前不開放外部使用。詳見 2-3 開放規劃。
項目 狀態 備註
後端 API 運行中 localhost:5002
前端操作介面 已完成 M4 中控台內建 Tax AI 頁面
公開網址 尚未部署 Phase 3 規劃後實施
手機 APP 未開始 React Native 或 PWA 待評估
SaaS 訂閱 規劃中 Phase 3 商業化方案
2-2  目前如何使用
推薦 方法 A：透過 M4 中控台

網址：http://localhost:5002/tax-ai

  • 選擇諮詢模式 → 輸入問題 → 取得 AI 回答（含法源引用）
  • 支援三種模式：諮詢模式 / 法規查詢 / 考試題型
方法 B：直接呼叫 API
POST http://localhost:5002/api/tax/query { "question": "外籍人士在台居留超過 183 天的稅務處理？", "mode": "consultation" } // 模式選項：consultation | query | exam
2-3  如何開放對外（Phase 3 規劃）
Step 1 — 後端雲端部署
部署至 Vercel（Serverless）或 Railway（容器），選定後可在 1 天內完成
Step 2 — 前端獨立建站
獨立網站或嵌入現有網站，提供乾淨的使用者介面
Step 3 — 使用者認證
免費方案：每日 5 題；付費方案：無限次查詢。Stripe 付款串接
Step 4 — Benchmark 達標後公開上線
目標：accuracy_avg > 0.85（82 道正式題目，55 選擇 + 12 申論 + 15 實務）才開放外部訪問
